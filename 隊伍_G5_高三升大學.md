## 2. 鐵則：做出「平板做不到」的東西（你的作品命脈）
平板/手機本身就有觸控、攝影機、麥克風、喇叭、網路、TTS、ASR——所以「辨識物件→語音回饋」「問答」「朗讀」「測驗」這類平板都能做，**拿不到創意與差異化分數**。
Kebbi 真正贏過平板、平板做不到的只有四件事，**你的主題就是靠這個**：
1. **移動**（前進/後退/旋轉）：在實體空間走位、靠近人、帶路、巡邏、佈置闖關。
2. **關節手勢**（頭/雙手）：比手勢、肢體示範、指向實物、身體語言。
3. **聲源定位 + 轉向**（方向性麥克風）：知道誰在說、在哪個方位，轉頭面對說話者。
4. **多機協作**（2 台以上 Kebbi）：多台在同一空間分工、對話接力、交棒走位——**最強的平板免疫**。

> 自我檢查：把你作品裡的「移動／關節／聲源定位／多機」拿掉後，如果平板還能達到一樣的成效，就代表不夠硬，要再往身體與空間靠。

#### 〔智慧教學〕雙機接力:歷史現場法庭辯論劇場

**對象與發展適配（為何適合這組年齡與程式深度）**
- 高三升大學（17-18 歲）已具備抽象思辨、多元史觀比較、與「論點 vs. 史實證據」的高階能力，能駕馭辯論的邏輯結構，而非只是看熱鬧；本作品把學習目標壓在「身體化的對峙臨場」上，正好用機器人的物理能力去做這個年齡層平板已經免疫的事。
- 程式深度落在中高階：需要兩台 Kebbi 的時序同步（多機協作協定）、聲源定位回傳角度後的轉向控制、關節手勢序列與台詞的對齊（speech-gesture sync），以及一台作為主控、一台作為從屬的事件交棒狀態機。對升大學生是有挑戰但可完成的工程量，能寫進備審作品集。

**真實需求/使用者故事**
- 高中歷史老師想讓學生「身歷其境」重現歷史爭議事件（如紐倫堡審判、馬關條約談判、伽利略宗教審判），但單向講述難以讓學生感受多方立場在同一空間裡正面交鋒的張力。
- 教師真正缺的不是「兩個論點的內容」（那確實平板能播），而是「兩個對立陣營在物理空間中分處兩端、互相逼近、轉身質詢陪審團」的對峙場域——讓學生在教室裡被兩股力量左右包夾，被機器人走近並轉頭點名要求出證。

**Kebbi 功能組合（標出平板沒有的能力）**
- 需要 **2 台 Kebbi**（核心機制即多機協作，缺一不可）。
- **【平板沒有：多機協作】** A 機=控方/甲陣營，B 機=辯方/乙陣營。兩台各載入立場知識庫與台詞，透過事件交棒（A 講完發出「交棒訊號」→B 接話反駁）形成真正的雙智能體接力，且各自佔據教室空間的兩端。
- **【平板沒有：移動】** 關鍵爭點時，兩機從兩端「物理走近彼此」到法庭中央正面對峙（逼近 = 衝突升溫），再各自「走向／轉向不同學生分組」蒐證；總結時退回兩端象徵立場定型。平板釘在腳架上無法走位逼近。
- **【平板沒有：聲源定位＋轉向】** 學生席舉手發言時，由發言所在的那一台 Kebbi 用方向性麥克風判斷聲源方位，**整台機器人轉身面向該名學生**收集論據，而非只在螢幕上切人臉框。
- **【平板沒有：關節手勢】** 攤手（表示「這毫無道理」）、伸臂指向對方機器人（指控）、指向實體證物或地圖、點頭/搖頭回應陪審團——用頭與雙臂做出辯論場上的身體語言。
- 其餘如 TTS 台詞、ASR 轉錄學生論點、投票計分屬平板也能做的「內容層」，在本作品中只作輔助，不作為差異化主張。

**運作流程（分步驟，具體可實作）**
1. 教師選定歷史爭議劇本，A 機載入甲方立場知識庫與台詞包，B 機載入乙方；兩機完成藍牙/區網握手與時間同步，分別走位到教室前方左、右兩端定位（移動）。
2. 開場：A 機面向陪審團陳述，比出強調手勢（伸臂、攤手），結束時對 B 機發出交棒訊號並轉頭面向 B（多機協作＋關節）。
3. B 機接收到交棒，轉頭面向 A 機反駁，伸臂指向 A 機做「指控手勢」，形成雙機物理對峙的接力對話。
4. 爭點升溫：兩機同時向教室中央移動、縮短彼此距離正面相對（移動製造逼近張力），點名請陪審團出證。
5. 學生分組舉手發言：由聲源側的那台 Kebbi 以聲源定位判斷方位、整機轉身面向該學生（聲源定位＋轉向），ASR 轉錄其論點並納入知識庫，點頭表示已採納。
6. 兩機根據學生新論據調整後續攻防（採納/反駁學生意見），再回到雙機接力辯論。
7. 總結陳詞：兩機退回兩端、面向全班做結辯手勢，引導學生投票；揭示真實歷史結果與多元史觀，並用手勢指向螢幕/海報上的史料對照。

**為何平板做不到-核心論證（差異化關鍵）**
- 本作品的學習目標已重新定位為「**在物理空間中被多方立場包夾、並被點名要求即時出證的對峙臨場感與責任感**」，而非「理解兩個論點的內容」。後者平板確實能做，前者平板結構性做不到。
- 拿掉物理性後測試：把兩台機器人換成左右雙螢幕＋立體聲，學生仍只是「旁觀兩段影片」——沒有任何一個實體會朝他走近、會因為他的發言而轉身把全身正對著他、會在爭點時逼近對手縮短物理距離。社會臨場感（social presence）研究指出，實體機器人的「朝向我轉身」與「向我靠近」會觸發真實的被注視壓力與參與責任，這是螢幕內人臉框切換無法產生的具身效應。
- 多機協作是最強的平板免疫：兩個獨立智能體分處空間兩端、各自轉頭、各自移動、向不同學生分組蒐證——這需要兩個物理本體在同一空間中佔據不同位置並協調動作，連兩台平板並排也只是兩塊不會動、不會轉向發言者、不會逼近彼此的螢幕，無法重現「兩造在法庭中央正面逼近對峙」的物理事件。
- 因此把核心機制設為「移動逼近 + 聲源定位轉身點名 + 雙機空間分置接力」三者綁定後，拿掉任何一項作品就垮：失去的不是娛樂感，而是這套作品賴以成立的具身對峙學習機制本身。

**對應評分亮點**
- 應用性：直接解高中歷史課「多方立場張力難傳遞」的真實痛點，劇本可換（審判/條約/革命），一套架構通用多單元。
- 創意性：把「法庭/談判桌」的物理對峙搬進教室，雙機分處兩端＋中央逼近的空間敘事，是平板無法複製的舞台設計。
- 挑戰性：雙機時序同步、聲源定位轉向、speech-gesture 對齊、學生論據即時納入的狀態機，工程含量高。
- 完整性：從劇本載入→接力辯論→學生蒐證→投票→史觀揭示，閉環完整且可重複上課。
- 展演：兩機走近對峙、轉身點名學生、同步退場結辯，現場視覺衝擊與互動性強，適合 6 分鐘舞台呈現。

**挑戰度與程式深度**
- 挑戰度：★★★★（4/5）。
- 程式深度：多機協作通訊協定（主從交棒狀態機、時間同步）、聲源定位角度→底盤/頭部轉向控制、台詞與關節手勢序列對齊、ASR 學生論點解析後動態插入辯論分支、投票統計與分支揭示。屬中高階整合專案。

**可現場 demo 的 MVP（先做最小流程）**
- 先做「**雙機接力 + 一次聲源定位轉身蒐證**」這條最小主軸：A 機陳述並交棒→B 機轉頭反駁→兩機向中央走近一步對峙→偵測一位舉手學生的聲源方位、由近側機轉身面向他收論點並點頭採納→兩機退回兩端做簡短結辯。先寫死一個劇本（如伽利略審判），把多機同步與轉向跑穩，其餘劇本與投票後加。

**展演高光時刻（6 分鐘最吸睛橋段）**
- 第 3-4 分鐘「中央逼近對峙」：兩機在台上同時從兩端向中央移動、距離縮到正面相對，A 機伸臂指向 B 機指控、B 機攤手反駁——兩個實體機器人在空間中真正面對面交鋒，這一刻平板永遠做不到。
- 緊接的「轉身點名」：評審/觀眾席有人舉手發言，近側 Kebbi 立刻整機轉身、全身正對發言者並點頭採納其論點，把全場注意力拉向被點名的人，製造強烈的具身臨場與互動驚喜。

#### 〔互動遊戲〕雙機接力說書劇場:歷史現場的多角色對戰問答

**對象與發展適配(為何適合這組年齡與程式深度)**
- 高三升大學(17-18歲)正處於抽象思辨、史觀建構與多元觀點權衡的黃金期,「主戰 vs 主和」這類無標準答案的歷史抉擇,恰好對接此年齡的批判性思維與辯論需求,而非低齡的記憶式問答。
- 程式深度匹配:本作需要雙機通訊同步(socket/MQTT 狀態機)、台詞接力的時序協調、聲源定位事件回呼、手勢與移動動作序列編排、投票即時統計與分支劇情狀態管理。對升大學程度的學生是「可挑戰但可完成」的綜合性專題,涵蓋網路通訊、事件驅動、狀態機三大核心觀念,有足夠的工程含量做為作品集。
- 跨域整合:歷史素養 × 機器人協作 × 互動設計,符合此組別跨學科統整、能自主分工(內容組/程式組/演出組)的成熟度。

**真實需求/使用者故事**
- 高中歷史老師想讓全班「身歷其境」重演歷史轉折點(如甲午戰前的主戰與主和、制憲時刻的聯邦派與反聯邦派)。單一螢幕只能單向講述,兩個歷史人物無法在空間中真正分立兩側對峙辯論,學生只能被動觀看、缺乏代入與選擇。
- 老師需要的是:讓兩個對立立場「活」在教室兩端,學生分隊喊話、投票,親身決定歷史走向,並在過程中理解「為何當時的人會如此抉擇」。

**Kebbi 功能組合(務必標出平板沒有的能力,需 2 台)**
- 需要 **2 台 Kebbi**,分立教室兩側。
  - **Kebbi A = 主戰派人物**;**Kebbi B = 主和派人物**。
- **多機協作(平板做不到 / 白名單5)**:兩台透過 socket/MQTT 通訊,做台詞接力與即時辯論交鋒、共享投票結果與當前劇情分支狀態,演出由雙機協調的「對手戲」。
- **關節手勢(平板做不到 / 白名單2)**:雙臂與頭部做激辯肢體——指向對方、攤手、握拳、搖頭,真實機械手臂的具身指向。
- **聲源定位 + 轉向(平板做不到 / 白名單3)**:用麥克風陣列判斷是左隊還是右隊在喊話,實體頭部/身體轉向發言陣營,做出「被一個身體注視」的回望回饋。
- **移動(平板做不到 / 白名單1)**:關鍵時刻兩機朝彼此前進逼近,以縮短物理距離製造對峙張力。
- *(平板也有的、不算特色的部分:語音 TTS 台詞、投票計分、分支文字內容——這些只是承載,非核心賣點。)*
- **協作分工**:A、B 各持半套劇本與立場資料庫;由其中一台擔任「主控/裁判機」維護投票計數與分支狀態並廣播,另一台為「跟隨機」依廣播狀態接話;每輪交鋒以「我說完→送出 cue→對方接話」的握手訊號確保不搶話、不冷場。

**運作流程(分步驟,具體可實作)**
1. **開場對峙**:兩台分立兩側,各自轉身面向對方,以手勢(握拳/指向)演出歷史衝突點,完成第一輪台詞接力對辯。
2. **拋出抉擇**:雙機向全班拋出歷史抉擇題(例:「我們該開戰還是議和?」),學生分左右兩隊。
3. **聲源定位回望**:某隊喊出主張,Kebbi 以聲源定位判斷方位,實體轉頭面向該隊,做出回應手勢與反問。
4. **投票分支**:兩隊投票(舉手由攝影機計數,或喊聲音量/按鍵),主控機統計後廣播結果,雙機接力演出對應的後續分支劇情。
5. **逼近製造張力**:劇情關鍵點,兩機朝彼此緩步移動逼近,配合升高的台詞與手勢,製造步步進逼的劇場壓迫感。
6. **結局回顧**:依累積選擇導向某個歷史結局,雙機轉向全班,複盤「真實歷史怎麼走、為何如此、若當初選另一邊會如何」。

**為何平板做不到-核心論證(差異化關鍵)**
- 本作核心成效押在「兩個實體智能體在共享物理空間中的物理對峙與協作演出」,這正是固定無臂平板無法平替之處:
  1. **物理空間對立感**:平板雖可分置兩側,但永遠「面向全班的螢幕」,不會把身體轉向對方、不會朝彼此移動逼近;立繪放大只是 2D 視覺把戲,缺乏「兩個實體在物理空間中縮短距離」帶來的臨場壓迫。
  2. **聲源定位回望**:真正依賴麥克風陣列 + 物理頭部轉向。智慧音箱聽得懂語音卻不會「轉頭看向喊話的那一隊」,平板朝向固定,失去「機器人察覺哪一隊在喊話並回望」這個即時、雙向、空間性的回饋。
  3. **關節手勢具身指向**:真實機械手臂指向對方、握拳,與螢幕內 2D 動畫的指向,在課堂臨場感上不等價。
- 三項中有兩項(聲源定位轉向、關節手勢)真實依賴機器人身體,加上多機物理移動逼近,構成最強的平板免疫。學習內容(辯論理解、投票、分支)雖可被平板承載,但**作品的核心賣點是「會動、會比手勢、會逼近、會轉頭的兩個實體角色之物理對峙」**——拿掉它作品就垮。

**對應評分亮點**
- **應用性**:直接服務歷史課「身歷其境重演轉折點」的真實教學痛點,可換主題複用(各時代辯論)。
- **創意性**:雙機對手戲 + 學生分隊投票即時分支,把課堂變成可參與的劇場。
- **挑戰性**:雙機通訊同步、台詞接力握手、聲源定位事件、移動逼近走位協調、分支狀態機,綜合工程難度高。
- **完整性**:從開場、互動、分支到結局回顧形成完整閉環,有教學前後與複盤。
- **展演**:兩機分立對峙、轉頭回望、相互逼近,具強烈舞台張力與觀眾代入。

**挑戰度與程式深度**
- **挑戰度:★★★★☆(4)**。
- 程式深度:雙機 socket/MQTT 即時通訊與狀態同步、台詞接力握手協定、聲源定位回呼觸發轉向、手勢/移動動作序列編排、投票即時統計、分支劇情狀態機。屬升大學程度可完成的高含量綜合專題。

**可現場 demo 的 MVP(先做哪一條最小流程)**
- 最小流程:**兩台分立兩側 → 完成一輪「台詞接力對辯」+ 一次「聲源定位轉頭面向喊話隊」+ 一次「投票決定一個分支」**。
- 先把雙機通訊與接力握手打通(這是地基),再疊聲源定位轉向,最後加投票分支;移動逼近與多分支可後續擴充。MVP 即可完整展示「平板做不到」的三大物理能力。

**展演高光時刻(6 分鐘最吸睛橋段)**
- **「步步進逼的對峙」**:辯論進入白熱化,兩台 Kebbi 一邊提高台詞與握拳手勢、一邊朝彼此緩緩移動逼近,在教室中央近身對峙——這是平板絕對重現不了、現場最有戲劇張力的一刻。
- 搭配前一秒的高光鋪墊:某隊學生大喊,兩台 Kebbi 同步轉頭「回望」該隊,讓觀眾感受到「被實體注視」的回應感,再轉回彼此進入逼近橋段。

#### 〔生活應用〕三機接力情境式失智長者懷舊互動劇場

**對象與發展適配（為何適合這組年齡與程式深度）**
- 高三升大學（17-18 歲）已具備抽象邏輯、跨領域整合與專題式探究能力，能勝任「多機協作 + 狀態機 + 感測整合」這種需要系統設計的高階題目。
- 本題涉及長照場域知識、非藥物療法（懷舊療法 Reminiscence Therapy）文獻、人因互動設計，正好對應這個年齡層「為真實社會問題提出解方」的成熟度與社會關懷。
- 程式深度落在進階：需要設計三機通訊協定、角色狀態同步、聲源定位事件觸發、走位路徑規劃與安全避障，是高年級組少數能撐起「平板免疫最強牌（多機協作）」的題型，挑戰性與展演性兼具。

**真實需求／使用者故事**
- 失智長照機構的照服員想用非藥物療法穩定長輩情緒，希望有會走動、會演的夥伴帶長輩進入懷舊生活情境（如菜市場、廟會），而不是看平板影片。
- 現場痛點：失智長者對平面螢幕投入度低、注意力難以維持；照服員人力有限，無法同時扮演多個角色、長時間陪伴一對一互動。
- 期望：用會移動、會比手勢、會轉頭回應的實體夥伴環繞長輩，重建「逛市場、被叫賣、被招呼」的臨場社會情境，並收集長輩反應供評估。

**Kebbi 功能組合（標明平板沒有的能力）**
- **需要 Kebbi 台數：3 台（多機協作，平板免疫最強牌）**。
- **【平板沒有｜多機協作】** 三台分飾不同角色，在同一實體空間分工演出、對話接力，連多台平板都無法重現「多個會動的實體智能體同時環繞長輩」的物理臨場。
  - **A 機（攤販機）**：主動移動靠近長輩、比手勢叫賣，是引發長輩趨近/伸手反應的主角。
  - **B 機（鄰居機）**：與攤販機對話接力（「老闆今天菜很新鮮喔」），製造熱鬧的社會聲景與多角色感。
  - **C 機（引導員機）**：帶長輩在空間中「逛攤」位移、銜接場景、收集反應，是流程節奏的指揮。
- **【平板沒有｜移動】** 攤販機「走過來」靠近、引導員機「帶長輩逛攤」位移、三機分散走位形成空間情境——平板釘在腳架上無法位移。
- **【平板沒有｜關節手勢】** 用頭與雙臂真實比劃叫賣、招手、招呼動作，提供具象動作線索供長輩模仿，喚起程序性記憶——平板只能播影片，無立體肢體存在感。
- **【平板沒有｜聲源定位＋轉向】** 長輩開口時，判斷方位並實體轉頭面向該位長輩，帶來「被注視、被回應」的社會臨場感——平板只能切螢幕、無物理轉向。
- （白名單外、平板也有的能力如 TTS 語音、ASR 辨識僅作輔助，不列為核心特色。）

**運作流程（分步驟，具體可實作）**
1. 照服員在引導員機（C）介面選定主題『菜市場』，C 機透過區網廣播場景指令給 A、B 機。
2. 三機依預設座標分散走位：A 機定位「攤位區」、B 機定位「鄰居區」、C 機停靠長輩旁。
3. A 機（攤販）規劃路徑、避障移動靠近長輩，舉雙臂比劃叫賣手勢並語音招呼。
4. 長輩回應出聲 → 啟動聲源定位，最近的機器人判斷方位並實體轉頭面向該位長輩，回應其話語。
5. A 機與 B 機進入對話接力腳本（攤販↔鄰居一來一往），推進「市場熱鬧」情境，並邀請長輩加入。
6. C 機（引導員）以低速、語音提示「我們去隔壁攤看看」帶長輩在空間中移動「逛攤」，A/B 機重新走位形成下一個攤位場景。
7. 全程記錄長輩反應（出聲次數、回應方向、互動時長）成簡易日誌，提供照服員評估情緒與投入度。

**為何平板做不到——核心論證（評審與差異化關鍵）**
- 本作的療效機轉建立在「移動 + 肢體 + 轉向」三項物理能力同時到位，缺一即垮：
  1. **主動移動靠近與走位**：攤販機繞行接近、引導員機帶長輩位移逛攤，誘發趨近、跟隨、伸手等身體反應，是固定音箱＋平面螢幕做不到的空間社會情境。
  2. **關節手勢的肢體示範**：比劃叫賣、招手能被失智長者直接模仿與回應（懷舊療法依賴具象動作線索喚起程序性記憶），純語音＋影片無法提供可被模仿的立體動作引導。
  3. **轉身面向說話者的具身回應**：聲源定位後實體轉向，帶來被注視、被回應的社會臨場感，遠強於音箱只是換邊出聲。
- 鎖定「對平面螢幕投入度低」的失智族群，三台實體角色在空間走動互動正是本作賣點；少了 Kebbi 的移動與關節能力，賣點被掏空、成效明顯改變。屬於少數「物理具身為核心、無法降級為螢幕/音箱方案」的情境，非包裝話術。

**對應評分亮點**
- **應用性**：直擊長照非藥物療法真實痛點，可進入機構場域實測，照服員可重複使用、可量測反應。
- **創意性**：把「多機分飾角色 + 走位 + 手勢叫賣 + 聲源轉向」組成空間實體劇場，懷舊療法 × 機器人協作的跨域整合。
- **挑戰性**：三機通訊同步、聲源定位事件觸發、走位避障與路徑規劃，屬高階系統設計。
- **完整性**：從照服員選題到反應收集評估，形成可閉環的服務流程。
- **展演**：六分鐘內三機走動、比手勢、轉頭，現場視覺衝擊強、敘事清楚。

**挑戰度與程式深度**
- 挑戰度：★★★★★（5）。
- 程式深度：進階。需多機區網通訊與角色狀態同步、聲源定位方位解算與轉向控制、移動避障與走位座標管理、對話接力狀態機、反應日誌記錄。

**可現場 demo 的 MVP（先做哪一條最小流程）**
- 最小流程＝「攤販機走近 → 比手勢叫賣 → 長輩出聲 → 聲源定位轉向回應」這一條單機核心鏈，先驗證移動 + 關節 + 轉向三能力串起來能跑。
- 第二步再加入 B 機對話接力（驗證雙機通訊），最後加 C 機帶路逛攤（驗證三機走位協作）。逐機疊加可控制風險、確保每階段都能展示。

**展演高光時刻（現場 6 分鐘最吸睛橋段）**
- 攤販機從遠處「主動走過來」靠近長輩（評審代演），雙臂高舉比劃叫賣手勢喊「來喔！今天的菜很青喔！」——移動 + 關節手勢同框最吸睛。
- 緊接長輩開口，機器人「啪」地實體轉頭面向他回應，同時鄰居機在另一側接話，兩機一來一往，引導員機緩緩帶長輩往下一攤移動——三機在空間中同時走動、比手勢、轉頭，呈現平板絕對無法重現的物理劇場高潮。
