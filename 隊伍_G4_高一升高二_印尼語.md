## 2. 鐵則：做出「平板做不到」的東西（你的作品命脈）
平板/手機本身就有觸控、攝影機、麥克風、喇叭、網路、TTS、ASR——所以「辨識物件→語音回饋」「問答」「朗讀」「測驗」這類平板都能做，**拿不到創意與差異化分數**。
Kebbi 真正贏過平板、平板做不到的只有四件事，**你的主題就是靠這個**：
1. **移動**（前進/後退/旋轉）：在實體空間走位、靠近人、帶路、巡邏、佈置闖關。
2. **關節手勢**（頭/雙手）：比手勢、肢體示範、指向實物、身體語言。
3. **聲源定位 + 轉向**（方向性麥克風）：知道誰在說、在哪個方位，轉頭面對說話者。
4. **多機協作**（2 台以上 Kebbi）：多台在同一空間分工、對話接力、交棒走位——**最強的平板免疫**。

> 自我檢查：把你作品裡的「移動／關節／聲源定位／多機」拿掉後，如果平板還能達到一樣的成效，就代表不夠硬，要再往身體與空間靠。

## 3. 你這隊的主題（含主攻 flagship 與備案，全文）

## 為何「學印尼語」不能只靠平板(差異化主張)
Duolingo 等平板語言 App 已能念印尼語、評發音、做對話測驗;因此本主題的價值必須來自 Kebbi 平板沒有的身體能力——**用關節手勢示範動作、在空間移動帶學生走位、用方向性麥轉頭面向發言者、兩台機器人在空間對話接力**。這些把「印尼語」與身體、空間、真人般的互動綁在一起,產生平板無法複製的沉浸與肌肉記憶。

> 以下第一個是**主攻主題**，其餘為備案/延伸。

#### Tebak Arah:聲源方位 × 印尼語方向詞的真人定向遊戲

**對象與發展適配**
- 對象:G4(高一升高二)獨立主題,程度設定為印尼語初中級。學生已有基本字母發音與簡單問答能力,可承接「方位詞 + 空間描述句」的進階口說。
- 發展適配:此年齡層具備抽象空間推理與「相對方位 vs. 絕對方位」轉換能力(我說的「右邊」與 Kebbi 的「右邊」是相反的),正適合用具身遊戲處理「視角轉換(perspective-taking)」這個語言+認知雙難點。多人輪替與兩兩裁判機制也吻合高中生偏好競賽、同儕互評的學習動機。

**真實需求/使用者故事**
- 身為學生,我希望透過「我說印尼語、Kebbi 真的轉向我並判斷我在哪個方位」來把方向詞(kiri/kanan/depan)學進身體裡,而不是在平板上點選方向選項。
- 痛點:傳統方位詞教學停在「圖卡上箭頭指左/右」,學生會背單字卻無法在真實空間中即時反應;尤其「以說話者為中心」與「以聽者為中心」的方位翻轉,平面練習完全練不到。

**會教到的印尼語內容(單字/句型 + 中文)**
- 方位詞:`di kiri`(在左邊)/ `di kanan`(在右邊)/ `di depan`(在前面)/ `di belakang`(在後面)
- 距離詞:`dekat`(近)/ `jauh`(遠)
- 自報位置:`Saya di sini!`(我在這!)/ `Saya di kanan.`(我在右邊。)
- 指令句:`Hadap saya`(面向我)/ `Putar ke kanan`(向右轉)/ `Putar ke kiri`(向左轉)
- 出題/裁判句型:`Siapa di sebelah kanan saya?`(誰在我右邊?)/ `Siapa yang di belakang?`(誰在後面?)
- 進階相對方位句:`Kamu di sebelah kiri saya, tapi di kanan Kebbi.`(你在我左邊,但在 Kebbi 的右邊。)— 練視角轉換

**Kebbi 功能組合(標出平板沒有的能力,需幾台)**
- 台數:**1 台 Kebbi**(G4 獨立設備,不併入跨組多機)。
- 【平板沒有・核心】**聲源定位(DOA, 到達方向偵測)**:用麥克風陣列計算分散就座的多名學生「真實聲音來自哪個方向」,作為方位詞對錯的真值(ground truth)。
- 【平板沒有・核心】**實體轉頭/轉身朝向**:Kebbi 物理正面旋轉指向特定學生,提供「機器人真的找到你了」的具身回饋,也作為反向題「被描述的對象」。
- 【平板可做・輔助】雲端 ASR 辨識方位詞、TTS 念題、LLM 出題與糾錯、計分。這些刻意定位為輔助層,拿掉硬體後遊戲機制即垮。
- 程式層:Python 計算 DOA 角度 → 與「座位地圖」比對 → 判定學生宣稱方位詞與物理 DOA 是否一致。

**運作流程(分步驟可實作)**
1. 開場校準:學生分散坐在 Kebbi 四周不同方位。每位學生先說一句 `Saya di sini!`,Kebbi 用聲源定位記錄各人 DOA 角度,建立「座位地圖」(角度 → 座位 → 方位扇區)。
2. 正向題(聲源驗證):Kebbi 不看畫面,只靠聲音出題 `Siapa di sebelah kanan saya?`。被指方位學生用印尼語回答 `Saya di kanan`,Kebbi 即時測該聲音 DOA,判定是否真的落在右側扇區,並轉頭面向驗證,口頭回饋 `Benar! Kamu di kanan.` / `Salah, kamu di depan.`
3. 反向題(實體朝向當刺激):Kebbi 隨機轉向某學生(實體正面旋轉),該生需正確說出自己相對 Kebbi 的方位詞,例如 `Saya di depan Kebbi.`;進階要求說出相對自身與相對 Kebbi 的差異。
4. 進階分組(裁判賽):兩兩一組互報方位,A 描述 B 的位置,Kebbi 用聲源定位 + 朝向核對「描述的方位詞」是否與「B 的真實聲源方位」一致,一致才得分。
5. 計分與糾錯:LLM 依錯誤類型給針對性回饋(例:把以說話者為中心的右誤用成以 Kebbi 為中心),累積積分排名。

**為何平板/Duolingo 做不到 — 核心論證(無懈可擊)**
- 真值來源不同:本設計把「你身體實際坐在哪個方位」當成答案的真值,真值由真人聲音的物理 DOA 提供。平板麥克風陣列不足以對教室中分散就座的多名學生做可靠 DOA 定位,只能退化成「螢幕上預先標好的頭像座標」或「學生自選按鈕」——把現實聲源退化成畫面選項,遊戲機制消失。
- 反向題不可複製:Kebbi 實體正面可旋轉、且只指向特定方位,成為「會變動、因人而異的物理刺激」。螢幕裝置無法在真實空間旋轉並指向個別學生的正面,反向題整體消失。
- 具身回饋不可複製:「轉頭面向你」把方位詞綁進真實空間的身體經驗;螢幕只能對全場呈現同一畫面,給不出「面向右後方的你」這種因人而異的空間回饋。
- AR 陀螺儀只能驗證持機者自身轉向,無法驗證房間裡另一個人發出的聲音來自哪個方向。
- 結論:拿掉聲源定位 + 轉向,只剩「念方位單字 + 點按鈕」,即落回 Duolingo 等平板 App 早已能做的範疇,作品核心垮掉。

**對應評分亮點**
- 應用性:真正解決方位詞「背得出卻反應不出、且不會做視角轉換」的教學痛點,可直接用於印尼語課堂。
- 創意性:以「真人聲源 DOA 作為語言對錯真值」的裁判機制少見,把語言判定綁進物理空間。
- 挑戰性:DOA 角度計算、座位地圖比對、ASR 方位詞辨識、即時轉向控制多模態整合。
- 完整性:出題 → 作答 → 物理驗證 → 轉向回饋 → 糾錯計分,閉環完整,正向/反向/分組三層次。
- 展演:Kebbi 閉眼只憑聲音轉頭精準指人,具強烈舞台張力與「機器人真的找到我」的驚喜。

**挑戰度與程式深度**
- 挑戰度:★★★★☆(4/5)。難點在 DOA 在多人、混響教室環境的穩定度,需做角度平滑與多次取樣;反向題的轉向控制與座位地圖維護也需工程整合。
- 程式深度與可串接 API:
  - 聲源定位 SDK / 麥克風陣列 DOA 取角 → Python 計算角度並映射座位扇區。
  - 馬達/頭部轉向控制 API(依角度旋轉朝向目標學生)。
  - 雲端 ASR(印尼語方位詞辨識,如語音轉文字服務)。
  - TTS(印尼語出題與回饋朗讀)。
  - LLM(出題生成、錯誤分類與針對性糾錯回饋)。
  - 計分/座位地圖狀態管理(本地 Python 資料結構)。

#### 鏡像體操教練:Kebbi 領做印尼語動作指令,攝影機評動作正確率

> 設計總綱:原版「Kebbi 做標準範本 → 攝影機比對學生關節」這條主線被兩位平板律師判定可被取代(真人預錄影片更標準、MediaPipe 平板原生可跑、Kebbi 無下肢無法蹲)。本展開版**捨棄「Kebbi 當人體姿態範本」這個劣勢主張**,把不可取代性重押在三件平板物理上做不到的事:**(1) 聲源定位 + 物理轉頭點名**——Kebbi 用頭部馬達轉向「正在用印尼語喊口令的那個學生」;**(2) Kebbi 在教室實體移動繞行**,把方位介係詞與動作詞綁進真實空間;**(3) 兩台 Kebbi 在三維空間互喊口令、互相轉向**,形成學生被夾在中間、必須用身體判斷「哪一台在對我下令」的空間化 TPR。攝影機評分仍在,但範本來源改為**人類教師預錄骨架**(承認律師說得對),Kebbi 的角色從「被模仿的範本」升級為「聽得到、看得到、會轉向、會走動的物理裁判兼領隊」。

- 身為高一升高二、想拿新南向獎項的學生,我發現「看影片學動作詞」最大的問題不是「誰示範得標準」,而是**課堂上沒人盯著我每一個人有沒有做對、沒人知道是誰在亂喊、沒人會走過來糾正我**。

**Kebbi 功能組合(標出平板沒有的能力,需要幾台 Kebbi)**
- 雲端 TTS 印尼語發音喊口令 —— 平板也有,**不算特色**(誠實標註)。
- 雲端姿態估計 API(MediaPipe / Google Pose / MoveNet)做學生骨架比對 —— 平板也能跑,**不算特色**(誠實標註);範本來源使用**人類教師預錄骨架關節序列**,放棄「Kebbi 當人體範本」的劣勢主張。
- **【平板沒有】聲源定位 + 頭部物理轉向**:Kebbi 麥克風陣列做 DOA(到達方向)估計,判斷「是哪個方位的學生在喊印尼語口令」,以**頭部馬達物理轉頭**面向發言者,完成點名 / 輪替。平板螢幕無法轉向、無法在物理空間指出「是你」。
- **【平板沒有】底盤實體移動繞行**:Kebbi 走到隊伍前方、繞行學生半圈、靠近做錯的學生,把 ke depan / mendekat / di sini 等空間詞綁進學生的本體感覺與真實距離。
- **【平板沒有,且為本案靈魂】雙機(2 台 Kebbi)空間化口令**:兩台 Kebbi 分站教室左右,各自喊不同印尼語口令並各自轉向不同學生,學生必須**先聽辨聲音來自哪一台、轉身判斷該台是否面向自己**,再決定要不要做動作(進階 Simon Says:只服從「面向你且喊 Tirukan saya 的那台」)。
- **【平板沒有】關節手勢領做**:Kebbi 用手臂 / 頭做出「上半身可對應」的動作(angkat tangan、putar kepala、tepuk tangan)當**節奏領隊與起拍信號**,而非當人體解剖範本——避開律師指出的「無下肢、無法蹲」弱點(jongkok / duduk 改由 Kebbi 喊口令 + 降低身體高度示意 + 真人骨架範本比對)。
- 台數:**2 台 Kebbi**(雙機口令對抗為核心高光;若僅 1 台仍可跑單機版,但雙機是本案不可取代性的關鍵升級)。

**運作流程(分步驟可實作)**
1. **暖身點名(聲源定位)**:Kebbi 喊「Siapa mau coba?(誰想試?)」,學生用印尼語喊「Saya!(我!)」;Kebbi 用麥克風陣列 DOA 算出方位,**物理轉頭面向該生**並說「Giliran kamu!(輪到你!)」。
2. **領做 + 比對(關節手勢 + 攝影機)**:Kebbi 喊「Ayo, berdiri!」並以手臂做起拍領隊動作;學生模仿;攝影機擷取學生骨架,系統計算學生關節角度與**人類教師預錄範本**的相似度(向量夾角差 + 關鍵點距離)。
3. **判定與回饋**:相似度過門檻 → Kebbi 說「Bagus!」並轉頭給下一位;未過 → Kebbi **走近該生(移動)**、放慢再領做一次並說「Coba lagi.」。
4. **空間化動作(移動)**:Kebbi 喊「Maju ke depan!(往前走!)」邊自己移動到隊伍前方,學生跟著走位,把方位詞綁進真實移動。
5. **雙機 Simon Says 對抗(2 台 Kebbi 高光)**:Kebbi-A 與 Kebbi-B 分站兩側,輪流或同時喊口令(如 A 喊 dua kali berdiri、B 喊 jongkok),但只有「轉頭面向你且先說 Tirukan saya」的那台才該服從;學生必須用聽辨 + 空間判斷做出反應,系統依攝影機評分 + 反應正確率(該不該動)計分。
6. **計分結算**:Python 彙整每位學生的動作正確率、反應速度、Simon Says 抗干擾正確率,輸出排行榜並由 Kebbi 用印尼語播報前三名。

**為何平板 / Duolingo 做不到 — 核心論證(無懈可擊)**
- **論證主軸不再依賴「Kebbi 當人體範本」**(此點已被律師正確駁倒,本案承認並繞開),改以三項平板物理上不可能的能力支撐:
- **(1) 聲源定位 + 物理轉向是空間社交動作,平板無此自由度**:平板的鏡頭與螢幕方向固定,無法在多人課堂中「轉頭指向正在用印尼語發言的那個人」。點名輪替(siapa / giliran kamu)在本案是**由物理轉向動作承載語意**——學生看到 Kebbi 的頭轉向自己才知道「是我」,這個「身體指向」是平板給不出的具身回饋。
- **(2) 實體移動把方位詞綁進本體感覺**:ke depan / mendekat / di sini 的學習成效來自學生**身體真的跟著一個會移動的領隊走位**並感受距離變化;平板只能在螢幕上播放箭頭,學生的前庭與本體感覺不參與,空間語意無法落地。
- **(3) 雙機三維空間口令對抗無法在單一螢幕重現**:Simon Says 的核心難度在於「**聲音來自空間中哪個位置、哪一台機器面向我**」——學生要用雙耳定位 + 轉身判斷才能決定服從與否。平板只有一個喇叭、一塊螢幕,無法製造「左前方那台對你下令、右後方那台是干擾」的真實空間聽辨情境。拿掉雙機與空間,這個機制直接垮成單純測驗,而那才是平板能做的。
- **結論**:本案把「學印尼語」的不可取代性建立在 Kebbi 的**聽(聲源定位)、走(移動)、轉(物理朝向)、雙機空間佈局**上,而非已被駁倒的「人體姿態範本」。拿掉這些身體與空間能力,作品退化為平板鏡像健身 App;保留它們,平板在物理上無法複製。

**對應評分亮點**
- 應用性:新南向印尼語零起點教學,TPR 全身動作詞 + 點名輪替 + 空間方位,課堂可直接落地、可拉低年級觀眾參與。
- 創意性:雙機空間化 Simon Says 抗干擾(該聽哪一台)是少見的「具身聽辨 + 服從判斷」遊戲機制。
- 挑戰性:聲源定位 DOA、雙機協調同步、攝影機多人姿態評分、Python 計分閉迴路四項整合。
- 完整性:感測(麥克風陣列 + 攝影機)→ 判斷(DOA + 姿態相似度 + 服從邏輯)→ 物理回饋(轉向 + 移動 + 領做)→ 計分結算,形成完整閉環。

**挑戰度與程式深度**

**可現場 demo 的 MVP**

**展演高光時刻**

- **轉頭點名**:Kebbi 聽到某位觀眾喊「Saya!」後當場物理轉頭面向他說「Giliran kamu!」,展現平板做不到的空間社交回饋。

#### Pasar 雙機殺價劇場:兩台 Kebbi 演市場買賣,學生當第三方議價手

**對象與發展適配**
- G4 高一升高二,獨立主題,使用 G4 自有的 2 台 Kebbi(顧客機 + 老闆機),不併入跨組多機彩蛋。
- 此年齡層已具基礎印尼語詞彙與句構,認知上能處理「語用層」(面子、軟化、語氣)而非只是字面翻譯;適合用「先觀察真實對話節奏、再被拉進三方競價」的高階口說任務。
- 三方競價的社會壓力(要搶在另一個買家前出價、又不能太衝撞老闆)對應高中生的社交語用發展,挑戰性與成就感兼具。

**真實需求 / 使用者故事**
- 學生想學會在印尼傳統市場 pasar 真實殺價:不是背句子,而是抓「節奏、語氣、面子拉扯」。
- 課堂上兩台 Kebbi 一台扮顧客、一台扮老闆,在攤位場景你來我往喊價、轉頭互相爭論;學生先看完一整輪真實議價節奏與面子角力。
- 看完後被老闆機邀請「換你來買」,學生以人聲加入,成為第三個來搶買的客人,與老闆機、顧客機形成三方競價;系統靠聲源定位判斷學生何時開口、轉頭面向他接話。
- 痛點:傳統 App 只能對著螢幕一對一練句子,學不到「真實市場裡兩個人互相轉頭、搶貨、給不給面子」的空間語用與輪替時機。

**會教到的印尼語內容(具體單字 / 句型)**
- 詢價:`Berapa harganya?`(這個多少錢?)
- 嫌貴:`Mahal sekali!`(太貴了!)
- 求降價(軟化策略):`Boleh kurang sedikit?`(可以便宜一點嗎?)
- 量議價:`Kalau beli dua, berapa?`(買兩個算多少?)
- 成交:`Ya sudah, saya ambil.`(好啦,我買了。)
- 殺價語氣詞:`Aduh!`(唉唷,表示嫌貴/為難)、`Wah`(哇,表示驚訝/誇張反應)
- 延伸還價句(老闆機 LLM 動態生成):`Tidak bisa, Pak/Bu.`(不行啦,先生/小姐)、`Harga pas.`(這是實價/不二價)、`Khusus buat kamu.`(特別算你便宜)
- 搶貨插話(顧客機):`Saya duluan!`(我先的!)、`Buat saya saja.`(賣給我就好)
- 面子軟化詞:`Pak`/`Bu`/`Mas`/`Mbak`(對攤商的敬稱)、`dong`/`ya`(口語軟化語助詞)

**Kebbi 功能組合(標出平板沒有的能力,需要幾台)**
- 台數:2 台 Kebbi(顧客機 + 老闆機),學生為第三方人聲加入。
- 【平板沒有】雙機對話接力:兩台實體 Kebbi 在物理空間中真實輪替喊價,各自轉向「對方」與「學生」——平板只能在單一螢幕上做動畫,無法呈現兩個獨立實體在桌面三個方位之間切換朝向。
- 【平板沒有】聲源定位 + 頭部轉向:同桌多名學生中,系統用麥克風陣列判斷「是哪個方位的學生開口」,老闆機實體轉頭面向該學生接話——平板預設「對著螢幕說話的就是當前使用者」,無法分辨同桌哪一位開口。
- 【平板沒有】底盤微移讓位:顧客機在學生加入時實體後退/側移,把「攤位前的買家位置」讓給學生,用空間位置標示「現在輪到你出價」——平板無法做物理讓位。
- 【平板也有,不算特色】雲端 LLM(動態還價策略與底線)、ASR(辨識學生喊的價格與議價句)、語用回饋計分——這些是底層,負責語言內容,但本作品的不可替代核心在上面三項身體能力。

**運作流程(分步可實作)**
1. 兩台 Kebbi 在攤位場景定位站好(老闆機在攤位後、顧客機在攤位前),顧客機與老闆機開始一輪標準殺價示範:邊喊價邊「實體轉頭」面向對方,顧客機嫌貴時老闆機轉頭回嗆,呈現語調與面子拉扯。
2. 兩機暫停,螢幕同步標出剛剛用到的議價句型(詢價→嫌貴→求降→量議→成交),並標記語氣詞 Aduh/Wah 的位置。
3. 老闆機說 `Sekarang giliran kamu!`(換你來買),系統開啟聲源定位;同時顧客機底盤微移後退讓位,空間上把買家位置交給學生。
4. 學生開口喊 `Berapa harganya?`;聲源定位判斷學生方位,老闆機實體轉頭面向學生,雲端 LLM 依市場行情生成還價與底線;學生用 `Mahal sekali! Boleh kurang sedikit?` 等句殺價。
5. 顧客機偶爾插話搶貨(`Saya duluan!`)形成三方競價,逼學生加快出價節奏與用對軟化詞;老闆機在兩個買家方位之間實體轉頭輪替應對。
6. 雲端記錄學生用了哪些句型、語氣詞是否到位、是否在顧客機搶走前議到目標價,給出語用回饋(太直接/沒給面子/該用 dong 軟化/成交價評分)。

**為何平板 / Duolingo 做不到-核心論證(無懈可教)**
- 學習核心不是「會不會講殺價句」,而是「在多人同桌的真實競價場中,抓對開口時機、判斷該轉向誰、用身體與位置爭買家身分」。這三件事的線索全部是物理空間的:
  - 對話輪替的視覺線索來自「兩個獨立實體在三個方位之間真實轉頭」,平板用單螢幕雙頭像動畫只是模擬,學生無法用自己的頭與身體方位去對應、跟讀真實的朝向切換。
  - 「現在輪到誰」由聲源定位 + 頭部轉向決定:同桌多名學生時,平板假設對它說話的就是當前學生,無法分辨方位,因此根本做不到「點到那個剛開口的人」的真實課堂議價輪替;這正是市場裡「老闆轉頭先理哪個客人」的核心語用。
  - 「搶買家位置」由顧客機底盤讓位 + 學生實際就位完成,把「我現在是買方」綁進身體與空間,平板無法做物理讓位。
- 反駁律師意見:律師主張語用核心由詞彙/語氣承載、轉頭只是臨場感。但「殺價語用」本質上就是「對誰說、何時插話、轉向誰」的多方互動時機判斷——把雙機與聲源定位拿掉後,三方競價塌縮成一對一句型練習,學生再也練不到「在另一個買家搶話前卡進去出價、且轉對人講對敬稱」這個最關鍵的可測殺價能力。因此身體能力不是裝飾,是承載「多方議價時機與朝向」這項學習目標的唯一載體。

**對應評分亮點**
- 應用性:直接對應赴印尼市場真實殺價情境,句型可立即帶出國使用。
- 創意性:把「三方競價搶貨」做成沉浸劇場,學生不是觀眾而是被拉進場的第三個買家。
- 挑戰性:雙機對話接力 + 聲源定位轉向 + 底盤讓位 + LLM 動態還價,多模態同步整合。
- 完整性:從觀察示範→句型標註→邀請加入→三方競價→語用計分回饋,閉環完整。
- 展演:雙機實體轉頭爭面子、機器人讓位、轉頭接話,舞台張力強、辨識度高。

**挑戰度與程式深度**
- 挑戰度:★★★★☆(4)。難點在雙機對話接力的時序同步、聲源定位與頭部轉向的延遲控制、三方插話的搶話時機。
- 可串接 API:雙機協作 / 多機同步指令、聲源定位(麥克風陣列 DOA)、頭部馬達轉向控制、底盤移動 API、ASR(印尼語語音辨識,含數字/價格擷取)、雲端 LLM(還價策略與底線生成 + 語用回饋評分)、TTS(印尼語語音合成,含 Aduh/Wah 語氣)、螢幕字幕/句型標註顯示 API。

**可現場 demo 的 MVP**
- 2 台 Kebbi + 一個簡易攤位道具(放 2 至 3 件商品)。
- 顧客機與老闆機跑完一輪 4 句殺價示範(含一次互相轉頭、一句語氣詞)。
- 老闆機說「換你來買」,顧客機底盤後退讓位。
- 學生喊 `Berapa harganya?`,聲源定位讓老闆機轉頭面向學生,LLM 還價一輪,顧客機插一句搶貨。
- 螢幕即時顯示學生用到的句型與一句語用回饋(例:「記得加 Pak 給面子」)。
- 最小可砍:LLM 還價可先用 2 至 3 條腳本化分支頂替,核心展示「雙機轉頭 + 聲源定位轉向 + 讓位」。

**展演高光時刻**
- 最吸睛橋段:三方競價中,學生喊出 `Mahal sekali! Boleh kurang sedikit?` 的瞬間,顧客機立刻插話 `Saya duluan!` 搶貨,老闆機在學生與顧客機兩個方位之間「實體連續轉頭」應對兩個買家,最後在學生喊出 `Kalau beli dua, berापa?` 加碼時老闆機轉頭面向學生點頭成交——三個角色在物理空間真的轉頭爭面子、搶生意,是平板雙頭像動畫永遠做不出的臨場張力。

## 決賽展演 & 直式影片致勝建議
決賽 = 直式影片（3–5 分鐘）＋ 現場 6 分鐘實體展演，影響「完整性 40 ＋ 展演 30」共 70 分。
- **直式影片**（手機直拍即可）：前 15 秒先講「誰、什麼痛點」（真實情境開場，別先講技術）→ 中段拍**真人實際使用 Kebbi、會動會比手勢**的一鏡到底 → 結尾講成效。字幕要大要清楚。
- **現場 6 分鐘**：一定要設計一個「**Kebbi 會動／轉頭／比手勢／多機交棒**」的高光時刻（Kebbi 閉眼只憑聲音轉頭、精準正面對準學生並用印尼語報方位）；務必準備**離線/單機降級備案**，避免網路或辨識失敗當場翻車。
- **分工**：1 人口述故事、1 人操作、1 人盯機器狀態。
