# 給 claude.ai 的討論 prompt — 旗艦 demo「輔導室陪伴機器人」的呈現方式

> 用法:把下面「===」之間整段貼到 claude.ai。它自足、不需額外檔案。

===

你是一位同時懂**教育科技產品 demo 策展**、**競賽簡報敘事**、以及**兒少心理/輔導倫理**的顧問。我要跟你深入討論一個比賽 demo 的「呈現方式」,請你給我多種路線、權衡、和具體建議,並且在倫理上幫我把關。請用繁體中文(台灣)。

## 我們做的東西

**輔導室陪伴機器人**,跑在 **Kebbi Air S**(NUWA 的桌上型教育機器人:有一張會動的臉螢幕、雙臂、麥克風、觸控感測、PIR 人體感測器;**不能移動**,只能轉頭/動手臂/做臉部表情)。

它的核心(都已實作、軟體測試全綠):
- **三層安全閘(確定性硬規則,不靠 AI 自由心證)**:每句學生的話先過規則比對 →
  - 🟢 **開放**:用真 LLM 陪聊,但**只傾聽、反映情緒、鼓勵多說,不分析、不深挖、不給建議**。
  - 🟡 **注意**(家庭/人際/長期低落/重大決定):不深挖,產一張**交接卡**進輔導老師待辦。
  - 🔴 **紅線**(自傷/受暴/危險):**立即呼叫現場真人(輔導老師)、停止 AI 對話**,並把完整脈絡交接給老師。鐵律:**寧可誤報、不可漏報**。
- **具身共情**(Air S 能做的):依燈號切**臉部表情**(暖/開心/傾聽/關切/**紅線時沉穩不驚恐**)、頭點頭/前傾、**PIR 感應到學生靠近就打招呼、離開降待命**、摸頭=正向回饋。
- **給輔導老師的工具**:逐句記錄落檔、一頁就懂的**結構化交接卡**(危急度、主要關注、輔導向度、建議層級)。

**定位(很重要)**:它**不是**取代輔導老師、**不做 AI 諮商**。它是「**降低學生開口的門檻 + 把危機更快、更完整地交到專業人手上**」的輔助工具。最有價值的動作就是「🔴 我去請老師過來陪你」。

**為什麼需要它**:學校輔導人力不足;很多學生(尤其**新住民子女、不擅言語、低齡、特殊需求**)不敢主動找輔導老師。一個溫暖、隨時在、會接住情緒、危機時立刻找真人的陪伴者,能補這個缺口。

**比賽取向**:評審重「**真的有實用性**」。

## 約束與敏感點(請一起幫我把關)
1. 這是**涉及兒少自傷風險**的題目 —— demo 不能消費/煽情真實痛苦,也不能讓人誤以為「AI 在做諮商/治療」。
2. **隱私**:會談記錄只落本機、**不存影像**;關鍵字內容最終要由輔導專業人員定稿(demo 用的是佔位)。
3. 不能**誇大**(例:宣稱能判斷/治療心理疾病)。它能做的就是「陪聊綠燈 + 確定性升級 + 交給真人」。
4. Live demo 有失敗風險(語音辨識、網路 LLM 偶發出錯)。

## 我想跟你討論的(請逐項給我有料的建議,不要只給原則)

1. **整體呈現路線**:給我 **2–3 種不同的 demo 呈現方式**(例:現場真人實演 vs 預錄+旁白 vs 情境劇+機器人輔助),各自的**賣點、風險、適合的評審類型**,最後**推薦一種**並說為什麼。
2. **紅線怎麼演**:要不要在台上實演「🔴 自傷風險」這一段?**倫理上**怎麼處理才不消費痛苦、不誤導?有沒有**替代手法**能展示「紅線即時呼叫真人」的機制,而不必演一段真實的自傷對話?(這題我最在意。)
3. **怎麼讓「具身同理」看得到、有感**:臉部表情、PIR 迎接、點頭、摸頭回饋——在台上怎麼安排才讓評審「感覺到它會接住人」,而不是看起來像花俏功能?
4. **怎麼凸顯「實用性」**:把「交接卡幫老師省力、確定性安全閘可稽核、寧可誤報」翻成評審聽得懂、信得過的呈現。
5. **對評審的敘事框架**:問題 → 洞察 → 方案 → 效果/證據,幫我擬一個有力的故事線 + **一句話定位**。
6. **時間分鏡**:假設只有 **5 分鐘** demo + 3 分鐘 Q&A,給我一個**逐段分鏡腳本**(每段秒數、台上發生什麼、誰說什麼)。
7. **失敗備援**:STT/網路當場炸掉時,怎麼設計讓 demo 仍走得下去、不開天窗。
8. **Q&A 防身**:列出評審最可能問的**尖銳問題**(例:「AI 判斷錯了害到學生怎麼辦?」「這會不會讓學校少請輔導老師?」「資料外洩?」),各給一個**誠實有力的建議答法**。

請先用 2–3 個釐清問題確認我的情境(比賽性質、評審背景、可用時間、是否能帶真機),再開始給建議。
