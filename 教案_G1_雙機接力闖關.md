# 初賽教案草稿 v1｜G1《雙機接力闖關：在教室地圖上跑的程式設計遊戲》

> **草稿 v1**（依建議主攻；待最終確認後微調）。對應大會初賽「教案格式」9 項、≤3 頁。評分對焦：應用性 35／創意性 35／挑戰性 30。
> 本教案的遊戲邏輯已用程式（`robot_app` 的 `RelayQuestGame`/`LevelMap`）實作並自測通過，可實機演示。

## 1. 科目領域
資訊科技（運算思維／程式設計）為主，融入數學（方向、座標、路徑最短化）與綜合活動（合作學習）。

## 2. 教學對象
國小五年級升六年級（10–11 歲），1 位學生獨立一組、搭配 1 位指導老師；程式深度以 **Blockly 積木式視覺程式**為主。

## 3. 學習目標
- **運算思維**：能用「順序 → 分解 → 條件判斷（IF 障礙）→ 迴圈」描述並解決一個空間任務。
- **雙機協調**：理解「任務分解」與「交棒同步條件」——A 機須抵達交接點，B 機才接手。
- **除錯（debug）素養**：能從「機器人撞牆／走錯」的物理結果反推並修正積木。
- **情意**：體會「我寫的指令 → 真實機器人在地板上動」的因果連結，建立程式設計的成就感與興趣。

## 4. 預期成果
學生能獨力以 Blockly 編出一段指令，讓 **兩台 Kebbi** 在教室地板的關卡圖上：A 機走位避障 → 抵達交接點交棒 → B 機接力繞過障礙抵達終點，並依「嘗試次數／撞牆次數／是否走最短路」獲得效率星等（★～★★★）。

## 5. 對應課綱核心素養（十二年國教）
- **資訊科技**：運算思維與問題解決——演算法、程式設計、除錯（資 E 相關）。
- **核心素養 A2 系統思考與解決問題**：將大任務分解為兩機子任務並協調。
- **核心素養 C2 人際關係與團隊合作**：雙機分工映射合作溝通。
- **數學 n/s**：方向、轉向角度、最短路徑（BFS 概念以遊戲星等具象化）。

## 6. 課程架構圖
```
第1節 認識 → 第2節 順序+避障 → 第3節 交接同步 → 第4節 闖關挑戰+反思
  ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
  │玩一次示範  │→ │ FWD/轉向  │→ │ HANDOFF  │→ │ 多障礙關卡 │
  │「平板做不到」│   │ IF_障礙   │   │ 站對交接點 │   │ 計分/星等  │
  │ 引起動機   │   │ 撞牆=失敗 │   │ 兩機交棒  │   │ 除錯反思   │
  └──────────┘   └──────────┘   └──────────┘   └──────────┘
```

## 7. Kebbi 教案設計（核心；標出「只有機器人做得到」）
**器材**：2 台 Kebbi、教室地板膠帶關卡圖（S 起點／G 終點／# 障礙／H 交接點）、Blockly 介面。

- **第 1 節・引起動機（撞牆 vs 繞行）**：老師先示範「直直往前」的指令 → A 機在地板上真的撞到障礙、闖關失敗。學生立刻看到「指令 → 物理結果」。〔**平板做不到**：真實地板走位與撞牆的空間因果〕
- **第 2 節・順序＋避障積木**：學生用 `FWD／LEFT／RIGHT` 排出繞行路徑；進階用條件積木 `IF_OBSTACLE…ENDIF` 讓程式自己偵測前方障礙再轉彎。撞牆即失敗 → 回去改積木（debug 閉環）。
- **第 3 節・雙機交棒同步**：引入交接點 H——**A 機必須先走到 H 才能 `HANDOFF`**，否則交棒失敗、B 機不啟動。A 機舉手比「換你！」＋語音，B 機回「收到，換我！」。〔**平板做不到（最強平板免疫）**：兩個獨立身體在同一片地板分工、會合、交棒〕
- **第 4 節・闖關挑戰＋反思**：挑戰多障礙「S 形強制路徑」關卡，系統依嘗試次數／撞牆次數／是否走最短路給 ★～★★★。學生反思「怎麼用最少步數過關」。
- **延伸（差異化）**：程度弱→只把 A 機走到交接點；程度好→挑戰 B 機避障與交棒時機。

**對應評分**：應用性（運算思維課堂可直接用、分層差異化）／創意性（指令變實體走位、雙機交棒）／挑戰性（條件積木＋兩機同步條件＋最短路徑優化）。

## 8. 教學成果與回饋
- **形成性評量**：每關以「是否抵達／撞牆次數／效率星等」即時回饋；學生填「我改了哪一步讓它過關」除錯紀錄單。
- **總結性評量**：能否獨立完成多障礙關卡並交棒成功；能否說明 IF 條件積木的作用。
- **預期回饋**：物理走位的即時因果大幅提升投入度與除錯耐受度（撞牆不再是挫折，而是線索）。
