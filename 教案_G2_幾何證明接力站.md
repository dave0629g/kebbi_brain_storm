# 初賽教案草稿 v1｜G2《幾何證明接力站：兩機分工帶讀證明的每一步》

> **草稿 v1**（依建議主攻；待確認後微調）。大會初賽 9 項格式、≤3 頁。評分：應用性 35／創意性 35／挑戰性 30。
> 遊戲邏輯已實作自測：`robot_app` 的 `GeometryRelayGame`（`--g2`/`--g2v`/`--g2h`）。

## 1. 科目領域
數學（幾何證明、邏輯推理）為主，融入資訊科技（進階積木／入門 Python 串接）。

## 2. 教學對象
國中二年級升三年級（13–14 歲），程式深度：進階積木 / 入門 Python。

## 3. 學習目標
- 能把一條幾何證明拆解為「已知 → 因為（推論）→ 所以（結論）」的邏輯層次。
- 能指認證明每一步對應的圖形元素（邊／角），建立「敘述 ↔ 圖形」的雙向連結。
- 能檢視一段證明腳本的**結構正確性**（缺步／順序錯置／層次標錯）並除錯。

## 4. 預期成果
學生站進地面放大幾何圖，兩台 Kebbi「**說—走—指—接棒**」：乙機念出每一步理由 → 甲機走到對應位置、**轉頭看學生**並用手臂指認該邊／角 → 回報接棒。學生先把證明步驟排成腳本，Kebbi 驗結構通過才放行接力。

## 5. 對應課綱核心素養
- **數學**：幾何性質的推理與證明（等腰三角形底角、三角形內角和 180°、外角定理）。
- **A2 系統思考與解決問題**：把證明分解為層次化步驟。
- **資訊科技**：以結構化腳本表達邏輯、結構驗證（除錯）。

## 6. 課程架構圖
```
第1節 證明的層次 → 第2節 邊角指認接力 → 第3節 學生自編腳本驗證 → 第4節 題庫挑戰+反思
 已知/因為/所以      乙機說→甲機走+指+轉頭      排步驟→驗結構→才接力      換題(內角和/外角)
```

## 7. Kebbi 教案設計（核心；標出「只有機器人做得到」）
**器材**：2 台 Kebbi、地面幾何放大圖、學習單／Blockly 或 Python 腳本介面。

- **第 1 節・證明的三層**：以等腰三角形「兩底角相等」為例，帶學生標出每步是「已知／因為／所以」。
- **第 2 節・邊角指認接力**：乙機念理由 → 觸發甲機**走到**地面圖對應位置 → **轉頭面向學生**（NeckZ；輪式機可底盤＋頭複合面向）→ 用**手臂指認**該邊／角 → 回報 DONE 換下一步。〔**平板做不到**：在地面放大圖上實體走位、關節指認、眼神跟隨被討論的圖素、雙機接棒〕
- **第 3 節・學生自編腳本驗證**：學生把證明步驟排成腳本，Kebbi 的結構驗證器先把關（缺步／多步／層次錯置／順序逆置／缺標層）→ 指出「第幾步、哪種錯」→ 改正才放行接力。〔即時結構除錯回饋〕
- **第 4 節・題庫換題＋反思**：換內角和、外角定理等題型，學生比較不同證明的層次結構。
- **差異化**：入門→照標準腳本接力；進階→自編腳本並通過驗證、挑戰多題型。

**對應評分**：應用性（數學證明課可直接用）／創意性（抽象邏輯外化成空間事件、雙機說走指接棒）／挑戰性（層次化＋結構驗證＋多題型）。

## 8. 教學成果與回饋
- 形成性：每步以「指認正確／接棒成功」即時回饋；學習單記錄各步層次與得分。
- 總結性：能否自編一段通過結構驗證的證明腳本並完成接力。
- 預期回饋：把「看不見的邏輯層次」變成「看得到的空間步驟」，提升證明的可理解度與參與度。
