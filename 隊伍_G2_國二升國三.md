## 2. 鐵則：做出「平板做不到」的東西（你的作品命脈）
平板/手機本身就有觸控、攝影機、麥克風、喇叭、網路、TTS、ASR——所以「辨識物件→語音回饋」「問答」「朗讀」「測驗」這類平板都能做，**拿不到創意與差異化分數**。
Kebbi 真正贏過平板、平板做不到的只有四件事，**你的主題就是靠這個**：
1. **移動**（前進/後退/旋轉）：在實體空間走位、靠近人、帶路、巡邏、佈置闖關。
2. **關節手勢**（頭/雙手）：比手勢、肢體示範、指向實物、身體語言。
3. **聲源定位 + 轉向**（方向性麥克風）：知道誰在說、在哪個方位，轉頭面對說話者。
4. **多機協作**（2 台以上 Kebbi）：多台在同一空間分工、對話接力、交棒走位——**最強的平板免疫**。

> 自我檢查：把你作品裡的「移動／關節／聲源定位／多機」拿掉後，如果平板還能達到一樣的成效，就代表不夠硬，要再往身體與空間靠。

#### 〔智慧教學〕幾何證明接力站:兩機分工帶讀證明的每一步

**對象與發展適配(為何適合這組年齡與程式深度)**
- 國二升國三(13-14歲)正值幾何證明的關鍵轉折期:從「直觀看圖」進入「演繹推理」,最常見的卡關正是分不清「已知 / 求證 / 理由」三層邏輯。
- 此階段學生已具備抽象符號運算能力,但仍高度依賴空間與動作記憶當鷹架,適合用「身體站進大圖、看機器人走位指認」把抽象邏輯外化成可追蹤的時空事件。
- 程式深度落在中高階:需處理「雙機狀態同步」「步驟接力排程」「移動座標 + 關節指向角度」,對國三生是「進階積木」級挑戰,但每一步都可用 Kebbi 積木式環境分段完成,難度可控、有延展性(可由學生自編走位順序)。

**真實需求/使用者故事**
- 國三生面對證明題常「知道答案大概長怎樣,但寫不出理由鏈」,把「因為兩邊相等」「所以三角形全等」的邏輯層次寫亂。
- 學生想要一個夥伴把證明「一步一步拆給我看」,而且不是螢幕上抽象高亮,而是「真的指到那條邊、那個角」,讓「這一步在講圖上的哪裡」變成看得見、跟得上的動作。

**Kebbi 功能組合(標明平板沒有的能力與分工)**
- **需要 2 台 Kebbi**,在同一張鋪在地面的放大幾何圖上分工接力。
- **甲機=「圖形指引機」**:核心用 **移動(白名單1)** 走到地面大圖對應的邊/角位置,再用 **關節手勢(白名單2)** 抬手臂、轉頭指向真實的線段與角,把「這一步講的是圖上哪裡」具象化。
- **乙機=「邏輯推理機」**:站在圖旁,負責宣告每一步的邏輯層次與理由(「已知…/因為…/所以…」),並用 **頭部點頭、手勢(白名單2)** 標記「現在進行到第幾步」。
- **兩機協作(白名單5,最強平板免疫)**:乙機說完一步理由 → 觸發甲機移動走位指認 → 甲機站定回報 → 乙機接著說下一步,形成「說—走—指—接棒」的空間化接力鏈。**移動 + 關節指向 + 雙機接力**三項都是平板(即使兩台架腳架)做不到的物理能力。

**運作流程(分步驟,具體可實作)**
1. 把一道幾何證明題的圖以巧拼/地墊放大鋪在地面,標好邊與角的格點座標。
2. 學生站進地面大圖中,手持學習單(欄位:已知 / 求證 / 理由)。
3. 乙機(推理機)宣告第一步:「因為 AB = CD(已知)」。
4. 觸發甲機(指引機)移動走到 AB、CD 兩條邊的位置,抬手臂依序指向,並轉頭看向學生。
5. 甲機站定回報「已指認完成」→ 訊號回傳乙機。
6. 乙機接棒講下一步理由(如「因為夾角相等」),甲機再走位指向該角。
7. 兩機一來一往接力,完成整條證明鏈;學生同步在學習單填寫每步理由。
8. 換題或進階:學生自行編排兩機的步驟順序與走位路徑(把證明邏輯「程式化」成走位腳本),反向驗證自己對證明結構的理解。

**為何平板做不到-核心論證(差異化關鍵)**
- 本作品的鷹架核心不是「高亮線段」,而是**身體尺度的空間具身**:學生站進放大圖中,用全身位置感把抽象邊角對應到腳邊真實空間,這是平板小螢幕無法提供的本體感與空間記憶。
- 實體機器人「走到」對應位置,讓指認帶有**路徑、方向、停頓的時間-空間敘事**——「下一步指向哪裡」變成可被身體追蹤的事件,而非螢幕上瞬間亮起的線段;這對「把邏輯步驟序列化、可追蹤」正是關鍵鷹架。
- 兩台機器在物理空間中分處不同位置、輪流走位接力,給出**「分工」的空間化具象**(一台在圖上動、一台在旁說),這種空間分離的雙角色臨場感,音箱雙聲道與螢幕分割無法等價。
- 誠實邊界(主動寫入提案以強化說服力):純「理解證明邏輯」的認知目標平板亦可達成。本作品的不可取代性,在於把學習鷹架**刻意綁定在具身空間經驗**,並用「學生自編走位腳本=把證明結構外化成可執行序列」這一設計,讓空間走位從「動機加分」升級為「對證明結構理解的可驗證輸出」——拿掉移動與雙機接力,這條鷹架即垮。

**對應評分亮點**
- 應用性:直擊國三幾何證明「邏輯分層卡關」這一真實痛點,配學習單可直接進課堂。
- 創意性:把「證明步驟」轉譯為「雙機空間走位接力腳本」,學生反向編排走位來驗證邏輯結構。
- 挑戰性:雙機狀態同步 + 移動座標 + 關節指向角度 + 接力排程,屬進階積木。
- 完整性:從鋪圖、接力、填學習單到換題自編,形成完整教學閉環。
- 展演:地面大圖 + 兩台機器人走位指認,空間敘事強、現場辨識度高。

**挑戰度與程式深度**
- 挑戰度:★★★★☆(4/5)。
- 程式深度:雙機訊息傳遞與接力旗標(handshake)、移動到指定座標的路徑控制、關節手臂指向角度計算、步驟序列狀態機;進階可開放學生自編走位順序(把證明鏈參數化)。

**可現場 demo 的 MVP(先做哪一條最小流程)**
- 先做「單題、固定 3 步、雙機一次接力」:乙機說理由 → 甲機走到該邊/角指向 → 回報 → 乙機接下一步,完成一條最短證明鏈(如等腰三角形兩底角相等的 3 步)。先確保**雙機接力訊號穩定 + 甲機走位指認準確**,其餘換題、自編走位為延伸。

**展演高光時刻(6 分鐘最吸睛橋段)**
- 第 4-5 分鐘:乙機朗聲宣告「因為這兩邊相等」後**靜默點頭示意**,甲機隨即**啟動移動、走過地面大圖**、停在那兩條邊上、**抬起手臂依序指向**再**轉頭望向觀眾與學生**——「說—走—指—回望」一氣呵成的雙機空間接力,觀眾能清楚看見「一台在動、一台在說」的分工臨場,是平板絕對複製不出的畫面。

#### 〔互動遊戲〕聲源捕手:循聲躲藏的空間反應遊戲

**對象與發展適配(為何適合這組年齡與程式深度):**
- 國二升國三(13-14歲)正處於抽象邏輯與空間推理快速成熟階段,能理解「方位角→轉向角度→移動向量」的座標與三角關係,也具備設計「狀態機」(倒數/偵測/鎖定/逼近/出局/換人)的程式抽象能力。
- 此年齡學生課業久坐、需要大肌肉群活動與團隊互動;本作是「全身閃躲跑動」的體感遊戲,兼顧運動量與社交,動機強。
- 程式深度可分層:基礎組做「聲源方位→轉向→直線逼近」的單回合;進階組加入難度遞增(縮短反應時間)、命中判定容錯、聲紋/音量門檻過濾雜訊,挑戰性足以支撐國中組競賽,又不會超出可實作範圍。

**真實需求/使用者故事:**
- 「想要動起來的學生需要一個會聽聲辨位、主動轉向甚至追過來的『鬼』,讓躲貓貓/反應遊戲有真實的空間追逐感。」
- 痛點:傳統躲貓貓需要一個「當鬼」的人犧牲遊戲樂趣;App 版的鬼只在螢幕裡,沒有「它真的朝我開過來」的物理張力。學生要的是一個公正、不會偷看、能在真實空間中循聲逼近的機器人鬼,讓「被靠近就出局」成為真實身體反應訓練。

**Kebbi 功能組合(平板沒有的能力已標注):**
- 需要 **1 台 Kebbi**。
- **【聲源定位 ★平板做不到】** 用方向性麥克風陣列判斷「拍手/喊口號」來自哪個方位(角度),而非只是「偵測到有聲音」。這是白名單第 3 類核心。
- **【轉向 ★平板做不到】** 鎖定方位後旋轉身體/轉頭面向發聲者,製造「它正盯著你」的臨場壓力。
- **【移動 ★平板做不到】** 驅動輪子穿越教室,朝發聲方位實際前進逼近,縮短與目標的物理距離。這是白名單第 1 類核心。
- **【實體在場 ★平板做不到】** 上述綜合成「會轉頭、會開過來」的實體鬼,帶來「它過來了,快跑!」的社會臨場感與物理閃躲互動。這是白名單第 4 類。
- 平板可做的部分(TTS 報數、音效、計分顯示)僅為輔助,不是核心機制。

**運作流程(分步驟,具體可實作):**
1. **開局佈陣:** 學生分散在教室各角落,Kebbi 置於中央。Kebbi 用 TTS 宣布規則並「閉眼」(螢幕表情)倒數 5-3 秒(難度越高倒數越短)。
2. **點名指令:** 系統隨機指定一種發聲指令(例如「拍手三下」或「喊口號」),由全體或被點到的人發出。
3. **聲源定位:** 麥克風陣列計算到達時間差(TDOA),解析出最強聲源的方位角 θ。
4. **轉身鎖定:** Kebbi 旋轉至面向 θ,並用表情/語音「我聽到你了!」製造鎖定壓力。
5. **物理逼近:** 驅動輪子朝該方位前進一段距離(可設安全速度與防撞),學生必須真實移動閃躲、逃離縮短距離的鬼。
6. **命中出局:** 當 Kebbi 逼近到設定的距離閾值(或前方感測偵測到人)即判定該方位最近者「出局」,出局者離場或成為協助裁判。
7. **回合升級:** 每回合縮短倒數與反應時間、提高移動速度,逐步加難度,直到剩最後存活者獲勝。

**為何平板做不到-核心論證(評審與差異化關鍵):**
- 本作核心是「判斷聲音來自空間哪個方向,並實際轉身、驅動輪子穿越教室朝那裡前進」的物理追逐。平板鎖在腳架上**不會位移、不會轉向**,只能把追逐降級成螢幕裡的虛擬鬼或榮譽制自我判定。
- 三項不可取代的物理機制:(1)**實體位移逼近**——沒有任何平板會從教室一端朝發聲者開過來,玩家就不需要真實閃躲逃離,身體反應與空間緊張感整個消失;(2)**轉身面向發聲方位**——平板/音箱無法旋轉指向「是你發出的聲音」,失去被鎖定的臨場壓力;(3)**真正的方位定位而非偵測**——消費級裝置與智慧音箱即使有麥克風陣列波束成形,也不對外輸出方向、更不會據此轉向移動。
- 結論:拿掉聲源定位與移動,剩下的只是一個會放腳步聲音效的計時偵測 App,已非同一作品。可被平板取代 = **false**。

**對應評分亮點:**
- **應用性:** 體育/反應訓練、班級破冰、課間活動皆可用,公正的機器人裁判取代「當鬼的人」。
- **創意性:** 把抽象的「聲源到達時間差定位」轉化為可被身體感知的「循聲追逐」遊戲機制。
- **挑戰性:** 即時方位解算、轉向角度換算、移動逼近與命中判定、難度遞增狀態機。
- **完整性:** 涵蓋開局→點名→定位→轉向→逼近→出局→升級的完整回合循環,可重複遊玩。
- **展演:** 「它真的轉過來朝我開過來」的物理橋段現場張力極高,觀眾一看即懂。

**挑戰度與程式深度:**
- 挑戰度 **★★★★☆(4/5)**。
- 程式深度:中高。需整合聲源定位 API(方位角讀取)、轉向角度控制、移動驅動與安全停止、距離/命中判定、回合難度狀態機、雜訊與音量門檻過濾(避免被無關聲音誤導)。

**可現場 demo 的 MVP(先做哪一條最小流程):**
- 最小流程:**單一發聲源 → 聲源定位取得方位 → 轉身面向 → 朝該方位前進固定距離 → 到達閾值喊「出局!」**。
- 先用單人拍手驗證「聽到→轉對方向→開過來」這條主軸能穩定運作,再疊加多人、倒數、難度遞增與雜訊過濾。

**展演高光時刻(現場 6 分鐘最吸睛橋段):**
- 高光:Kebbi「閉眼」倒數後,場上一名學生拍手,Kebbi **猛然轉頭面向他、表情變「鎖定」、語音「我聽到你了!」並驅動輪子朝他開過去**,該生驚呼閃避逃跑——這個「轉頭+開過來」的瞬間,把聲源定位、轉向、移動、實體在場四項平板免疫能力一次展現,全場最有張力。

#### 〔生活應用〕走廊節能巡邏隊:校園能源浪費實體稽查員

**對象與發展適配**
- 本主題鎖定國二升國三(13-14 歲)，此階段學生已具備抽象邏輯與系統化思考能力，能處理「條件判斷 + 狀態機 + 資料累積」的中階程式結構，但尚未到需要完整演算法理論的程度，難度落點剛好。
- 主題切合此年齡的真實校園角色(環保股長、值日生、班級幹部)與公民責任感的發展任務，學生有強烈的「我在替全班/全校做事」的動機。
- 程式深度可分層:基礎組做「巡邏路線 + 偵測 + 手勢提醒」，進階組加「資料累積 + 每日熱點報表 + 條件閾值調校」，方便差異化教學與分工。

**真實需求/使用者故事**
- 身為負責教室節能的環保股長，我想在下課時派 Kebbi 在走廊與各教室間巡邏，主動走到「沒關燈關扇卻沒人」的角落並比手勢提醒，因為人工逐間巡查費時、容易漏掉，且同學常覺得「被人盯」尷尬，由機器人來指認比較中性。
- 痛點具體且可量化:校園下課/午休時段的「亮燈無人」「電扇空轉」是真實且可被偵測的能源浪費，成果可用「每日熱點清單 + 減少浪費分鐘數」直接驗收。

**Kebbi 功能組合(標示平板沒有的能力)**
- 需要 Kebbi 台數:**1 台**(單台即可完成核心使用者故事;此主題不需多機協作，刻意保持精簡以提高完整性與可實作性)。
- **【移動・平板沒有】自走巡邏與主動靠近**:Kebbi 依預設走廊路線前進/旋轉巡邏，偵測到浪費點後**主動移動靠近**該角落並轉向面向浪費點(含基本避障)。平板架在腳架上做不到。
- **【關節手勢・平板沒有】雙手肢體指認**:抵達現場後用頭與雙手做出「指向那一盞燈/那台電扇」的物理指向手勢，搭配「請關燈」的提醒動作(搖手、雙手向下示意關閉)。這是平板螢幕箭頭無法等價的**現場物理指認**。
- **【實體在場・平板沒有】臨場稽查壓力**:一個會走到你面前、轉身、舉手指認的實體稽查員，對路過同學產生直觀的提醒與行為矯正效果。
- 平板「也做得到」、不算特色的部分(誠實標示):光照/聲音偵測、語音播報、時間地點記錄、報表彙整——這些是輔助層，**不是**作品賣點，賣點是「移動到現場 + 肢體指認 + 實體臨場」三項物理能力。

**運作流程(分步驟、可實作)**
1. 路線初始化:在走廊地面設定固定巡邏路徑(以時間/編碼旋轉組合走「定點 A→B→C」折返),每個定點對應一間教室或一處燈具/電扇。
2. 定點偵測:Kebbi 到達定點後停下，讀取光照感測(亮)與麥克風音量/動靜偵測(連續 N 秒無人聲/無動靜),兩者同時成立 → 判定為「疑似浪費點」。
3. 主動靠近:判定成立後，Kebbi 旋轉並向該浪費點方向前進一小段(靠近燈具/電扇),製造「走到現場」的在場感。
4. 肢體指認:轉向目標，用單臂/雙臂做出指向手勢，頭部朝向目標，語音同步播報「這裡的燈/電扇開著但沒人，請記得關閉喔」。
5. 記錄:把時間、定點代號、浪費類型(燈/扇)寫入累積清單。
6. 回到路線:完成提醒後返回巡邏路徑，繼續下一定點。
7. 日終報表:巡邏結束後，輸出「今日浪費熱點清單」(哪個定點被觸發最多次/累積無人亮燈分鐘數),交給環保股長分析、貼班級公告。

**為何平板做不到-核心論證**
- 使用者故事的動詞是「**派它去巡**」與「**走到那個角落比手勢**」,核心機制是「在實體空間中移動到現場 + 用肢體指認實體燈具」。拿掉移動與關節手勢，作品只剩一台不會動的定點偵測器，使用者故事直接垮掉。
- 平板架在腳架上**無法自行移動**到不特定的浪費點;若要覆蓋全校得佈很多台平板/感測點，那已從「一台會巡的稽查員」變質成「固定感測網」,**改變了作品本質**，且仍補不回「移動到現場」這件事。
- 平板螢幕只能顯示箭頭或唸出教室名稱，**無法在物理現場把「就是這盞燈」用手臂具體指出來**;Kebbi 的關節指認對任何路過、沒看螢幕的同學都直觀，這是螢幕箭頭不可替代的物理指向。
- 「一個稽查員真的轉身走到你面前舉手指認」的臨場壓力與行為矯正，是實體在場的社會效果，螢幕無法產生。
- 誠實揭露的反面與我們的設計回應:偵測準確度與報表確實感測網更便宜更強——所以我們**不把作品價值押在偵測/報表**(它們只是輔助),而是明確定位於「移動到現場 + 肢體指認 + 臨場提醒」這層平板與感測網都做不到的物理教育價值。評審若問「為何不用感測網」,標準回答是:本作品要的不是最便宜的偵測，而是「會走到現場、用身體指給你看」的稽查體驗，這正是 Kebbi 獨有。

**對應評分亮點**
- 應用性:對準真實校園節能任務，成果可量化(熱點清單、減少浪費分鐘數),能直接落地班級使用。
- 創意性:把抽象的「節能規勸」轉成「會走到現場、用手指燈」的實體稽查員角色，新穎且有記憶點。
- 挑戰性:整合移動巡邏 + 多感測條件判斷 + 關節手勢 + 資料累積報表，是中階整合題。
- 完整性:單台、流程閉環(巡邏→偵測→靠近→指認→記錄→報表),不靠多機，容易做完整、好驗收。
- 展演:有明確「走過去 + 轉身 + 舉手指燈」的高張力畫面，舞台效果強。

**挑戰度與程式深度**
- 挑戰度:**★★★☆☆ (3/5)**。
- 程式深度:狀態機(巡邏/偵測/靠近/提醒/記錄/返回)+ 雙感測融合條件判斷(光照 AND 靜音時長閾值)+ 路線編碼移動與旋轉 + 關節手勢序列 + 累積資料結構與日終報表彙整。閾值調校(避免有人卻誤判)是主要難點與得分點。

**可現場 demo 的 MVP(先做哪一條最小流程)**
- 最小可行流程:設「2 個定點 + 1 段直線走廊」。Kebbi 走到定點 A → 偵測到「亮 + 無動靜」→ 旋轉並前進靠近 → 轉向燈具方向舉手指認 + 語音「請關燈」→ 記錄一筆 → 返回。
- 先把「偵測成立 → 移動靠近 → 手勢指認」這條物理鏈打通(這是平板免疫的核心),報表先以最簡單的計數清單呈現，行有餘力再美化。

**展演高光時刻(現場 6 分鐘最吸睛橋段)**
- 高光在第 3-4 分鐘:現場故意留一盞「亮著但沒人」的燈(道具燈),Kebbi 巡邏經過、停頓偵測、**轉身、緩緩走近那盞燈、抬起手臂明確指向它**，同時語音「這裡的燈開著卻沒人，請記得關燈喔」——「走過去 + 轉身 + 舉手指認」這組移動加關節的連續動作，是平板絕對演不出來的畫面。
- 收尾高光:Kebbi 回到台前，頭部轉向環保股長(評審席方向),口頭播報「今天抓到 3 處浪費熱點」,把實體稽查與資料報表收成一個完整故事。

## 備案與評審提醒
- 本隊主檔含 3 個主題：**主攻＝幾何證明接力站（評審認證最具差異性之一，具身幾何）**，另含「聲源捕手（循聲躲藏遊戲）」「走廊節能巡邏隊（生活應用）」。
- ⚠️ 「走廊節能巡邏隊」評審指出**平板免疫偏弱**（固定感測器也能偵測亮燈無人）；若要採用，必須把核心放在「**移動稽查路徑規劃 + 現場肢體指認 + 對人即時勸導**」，並說明為何固定感測網不可取代（如需進入無布線的臨時空間、需與現場人互動）。

## 決賽展演 & 直式影片致勝建議
決賽 = 直式影片（3–5 分鐘）＋ 現場 6 分鐘實體展演，影響「完整性 40 ＋ 展演 30」共 70 分。
- **直式影片**（手機直拍即可）：前 15 秒先講「誰、什麼痛點」（真實情境開場，別先講技術）→ 中段拍**真人實際使用 Kebbi、會動會比手勢**的一鏡到底 → 結尾講成效。字幕要大要清楚。
- **現場 6 分鐘**：一定要設計一個「**Kebbi 會動／轉頭／比手勢／多機交棒**」的高光時刻（一台在地面大圖走位指認、一台在旁宣告理由的雙機接力）；務必準備**離線/單機降級備案**，避免網路或辨識失敗當場翻車。
- **分工**：1 人口述故事、1 人操作、1 人盯機器狀態。

- **本隊技術實作路線(系統架構/關鍵 SDK 呼叫/繞授權牆 workaround/單機降級備案)**：見 `技術實作路線_各隊.md` 中本隊小節。
