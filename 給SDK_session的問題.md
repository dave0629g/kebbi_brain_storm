# 給 Kebbi Unity SDK session 的提問 prompt（可直接複製貼上）

---

我要用 **Unity + NuwaUnity SDK** 為 **NUWA Kebbi Air / Air S** 開發機器人 App，請以你對 Kebbi Unity SDK 的了解逐題回答。**每題請盡量附上對應的 class / method / event 名稱、範例程式片段，或官方文件連結；不確定的請明確標「不確定」**，不要猜。

## A.〔最關鍵〕聲源定位 / 麥克風陣列
1. Unity SDK 能不能**用程式讀到「聲音來自哪個方向」**（DOA / sound source localization / 聲源方位角度）？是哪個 class / event / API？回傳的是**角度（度數）還是方位扇區**？請給範例。
2. 這個方向資訊只在「喚醒詞 / 語音辨識」時有，還是**任何聲音（拍手、喊叫）都能取得方向**？
3. 有沒有內建的「**自動轉頭/轉身面向說話者**」行為可以直接觸發？觸發後我能不能讀到它轉向的角度？
4. 角度解析度/準確度大概多少？**機器人正後方的聲音**也能定位嗎（360°）？
5. （硬體確認）Kebbi Air S 的 6 麥克風陣列聲源定位，是只用於系統內建行為，還是有開放給開發者的 API？

## B. 底盤移動
6. 移動的 API 有哪些：**前進/後退指定距離、原地旋轉指定角度、移動到相對座標**？方法名稱？
7. 有沒有**里程計回授**（實際走了多遠、目前朝向角度）？有沒有避障？能做到「走到地面上指定的點」這種可重複的精準走位嗎？

## C. 關節（頭 / 手臂）
8. 能不能**直接設定頭與手臂的關節角度**（不只是播放預設動作）？自由度與各關節角度範圍？
9. 能不能播放**自訂的逐幀動作序列並控制節拍/停留時間**（例如體操逐拍示範）？
10. 能不能讓手臂**指向某個指定方向/角度**？

## D. 攝影機 / 視覺
11. Unity 裡能不能**取得原始相機影像**（camera frame / texture）讓我跑**自己的電腦視覺**（如 MediaPipe 姿態偵測）？還是只能用 NUWA 內建的 FaceTrack / Face_Recognize？
12. 有沒有**人體姿態（pose / skeleton）偵測**，還是只有臉部？能不能從影像取得「人相對機器人的方位/距離」？

## E. 語音 ASR / TTS 與語言
13. 內建 ASR 與 TTS **支援哪些語言？特別是「印尼語（Bahasa Indonesia）」有沒有支援？**
14. 若不支援印尼語：Unity 能不能**關閉內建語音、取得原始麥克風音訊串流**送到**外部雲端（Google/Azure 印尼語 STT/TTS）**，再用喇叭播放我自己的音訊？取得 mic audio buffer 的 API 是？

## F. 網路 / LLM
15. Kebbi 上的 Unity App 能不能**自由對外發 HTTPS 請求（UnityWebRequest）呼叫任意外部 API（例如 LLM API）**？有沒有網路沙箱 / 白名單 / 防火牆限制？
16. SDK 本身有沒有內建 **GenAI / LLM 模組**？使用它**是否需要 RaaS / GenAI 授權**？沒有授權時還能用嗎？

## G. 多機協作（兩台以上 Kebbi）
17. SDK 有沒有官方的**多機 / 機器人對機器人通訊**機制？
18. 若沒有：兩台 Kebbi 各跑一個 Unity App，能不能在**同一 Wi-Fi 下用 socket / MQTT / 藍牙互通**？App 能否在背景跑網路服務？有無限制？

## H. 部署環境
19. 完整工具鏈：**Unity 版本、NuwaUnity SDK 版本、Android build target**、如何把 App 部署/側載到 Kebbi、如何在實機 debug？
20. SDK 主要在哪一型（Kebbi Air / Air S）驗證過？**不同型號在上述能力（尤其麥克風陣列 DOA）有差異嗎？**

## I. 文件 / 範例
21. 上述每一項，請給對應的**官方文件頁面或 GitHub 範例**連結（特別是：DOA 聲源方向、底盤移動、關節角度、相機影像存取、對外網路、多機）。

---

> 回答完後，我最需要確定的三件事是：**(1) DOA 角度能不能用程式讀到、(2) 印尼語語音支不支援（或能否走外部雲端）、(3) 兩台 Kebbi 能不能自寫網路互通**。這三項請務必明確回答。
