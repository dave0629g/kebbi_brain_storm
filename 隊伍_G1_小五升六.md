## 2. 鐵則：做出「平板做不到」的東西（你的作品命脈）
平板/手機本身就有觸控、攝影機、麥克風、喇叭、網路、TTS、ASR——所以「辨識物件→語音回饋」「問答」「朗讀」「測驗」這類平板都能做，**拿不到創意與差異化分數**。
Kebbi 真正贏過平板、平板做不到的只有四件事，**你的主題就是靠這個**：
1. **移動**（前進/後退/旋轉）：在實體空間走位、靠近人、帶路、巡邏、佈置闖關。
2. **關節手勢**（頭/雙手）：比手勢、肢體示範、指向實物、身體語言。
3. **聲源定位 + 轉向**（方向性麥克風）：知道誰在說、在哪個方位，轉頭面對說話者。
4. **多機協作**（2 台以上 Kebbi）：多台在同一空間分工、對話接力、交棒走位——**最強的平板免疫**。

> 自我檢查：把你作品裡的「移動／關節／聲源定位／多機」拿掉後，如果平板還能達到一樣的成效，就代表不夠硬，要再往身體與空間靠。

#### 〔智慧教學〕雙機接力闖關:在教室地圖上跑的程式設計遊戲

**對象與發展適配(為何適合這組年齡與程式深度)**
- 10-11 歲(五升六)正處於皮亞傑「具體運思」轉向「形式運思」的關鍵期:抽象的程式邏輯(順序、迴圈、條件、分解)需要先掛在「看得到、摸得到、踩得到」的具體事物上才好內化。讓 Blockly 指令直接變成地板上機器人的真實走位,正是把抽象概念「具體化」的最佳鷹架。
- 程式深度設定在「順序 → 分解 → 條件判斷 → 兩機協調(交棒同步)」,難度由淺入深、可分層:程度弱的學生先把 A 機走到交接點;程度好的學生挑戰 B 機避障與兩機交棒時機的同步條件,符合差異化教學。
- 兩台分工天然引導「任務分解」與「團隊溝通」,呼應高年級的合作學習與運算思維(Computational Thinking)課綱目標。

**真實需求/使用者故事**
- 「我想邊玩邊學程式,設計一個兩台 Kebbi 在地上分工合作、交棒移動去『闖關取寶』的遊戲,讓寫 Blockly 的指令真的變成機器人在空間裡的走位。」
- 痛點:傳統螢幕迷宮遊戲裡,學生看到的是 sprite 在畫面上動,指令與「物理結果」之間隔了一層螢幕,因果感薄弱;撞牆只是畫面停住,沒有真實衝擊與空間感。學生想要的是「我寫的這行指令,讓那個真的會動的機器人,在我腳邊的地板上,真的往前走了三步」。

**Kebbi 功能組合(標出平板沒有的能力)**
- 需要 **2 台 Kebbi**。
- 〔白名單1・移動〕**平板做不到**:兩台都需在真實教室地板上前進/後退/旋轉,沿膠帶關卡走位、避開實體障礙、抵達物理交接點與終點。平板架在腳架上無法自走。
- 〔白名單5・多機協作〕**平板做不到(最強平板免疫)**:A、B 兩台在「同一片真實地板」分工、定位、會合、交棒——兩個獨立身體在共享物理空間協調位移,這是螢幕雙 sprite 無法等價的。
- 〔白名單2・關節手勢〕**平板做不到**:A 機抵達交接點時舉手比「換你」手勢、終點兩機一起做勝利動作。平板沒有手臂。
- 分工說明:
  - **Kebbi A(前鋒)**:從起點出發,依 Blockly 路徑走完前半段,抵達交接點後停下、舉手做「換你」手勢並可語音示意。
  - **Kebbi B(接力者)**:在交接點待命,偵測到 A 的交棒訊號(手勢/語音/藍牙或定時同步)後啟動,走完後半段、避開障礙、抵達寶物與終點。
  - **協作方式**:以「交接點」為兩機的同步契約——A 的「完成」即 B 的「啟動條件」,學生需用條件邏輯設計交棒時機,兩機在終點同步觸發勝利手勢。

**運作流程(分步驟,具體可實作)**
1. 學生用彩色膠帶在地板上佈置關卡:起點、轉彎、實體障礙(如紙箱/三角錐)、寶物、交接點、終點。
2. 用 Blockly 撰寫 A 機路徑指令(前進 N 步、左/右轉、到交接點停)。
3. A 機實際沿膠帶走到交接點停下,舉手並喊「換你」。
4. B 機接收交棒訊號後啟動,走完剩餘關卡、避開障礙、取得寶物、抵達終點。
5. 兩機在終點一起做勝利手勢(舉雙手/點頭)+ 勝利音效。
6. 若撞到障礙或走錯 → 視為闖關失敗,回頭除錯改 Blockly,即時看到「指令修改 → 物理結果改變」的因果回饋。

**為何平板做不到-核心論證(差異化關鍵)**
- 本作品的核心學習價值是建立「**Blockly 指令 = 一個實體在你能踩到、能繞著走的真實空間裡產生位移**」這條因果連結。平板是定點裝置:它不能在地板上自己前進/轉彎/避障,沒有手臂舉手示意,也不能做肢體勝利動作。
- 撞到實體障礙的「即時物理回饋」、群體圍觀同一物理事件的臨場感、兩個獨立身體在共享空間的會合與交棒,都是螢幕雙 sprite 無法等價的空間感與協調。
- AR 雖能把虛擬機器人疊到地板,但移動的是螢幕裡的影像而非可觸摸的物體,失去「實體在場且自己會動」的具身感。
- 拿掉移動與第二台,作品就退回成螢幕上的迷宮小遊戲,核心價值整個喪失——這正是兩位平板律師判定「可被平板取代=false」的理由。要靠純平板補足,只能外接會動的輪型小車,等於承認需要一個會動的機器人替身。

**對應評分亮點**
- **應用性**:直接對接運算思維課綱(分解/順序/條件/協調),可作為國小程式課的實體教具,複用性高。
- **創意性**:把「程式碼=螢幕」翻轉成「程式碼=地板上的實體接力賽」,雙機交棒是少見的具身化設計。
- **挑戰性**:兩機在共享空間的同步交棒、B 機避障條件邏輯,技術門檻明顯高於單機定點任務。
- **完整性**:從佈置→寫碼→執行→除錯→勝利,具備完整閉環與失敗重試機制。
- **展演**:雙機在地板上接力走位、舉手交棒、終點同步勝利手勢,現場觀眾可圍觀同一物理事件,張力十足。

**挑戰度與程式深度**
- 挑戰度:★★★★☆(★4)。
- 程式深度:順序指令 → 任務分解(前/後半段) → 條件判斷(避障、交棒觸發條件) → 兩機同步協調(A 完成為 B 啟動條件)。可分層出題,進階組挑戰交棒時機的條件同步與避障演算。

**可現場 demo 的 MVP(先做哪一條最小流程)**
- 最小流程:**單一直線路徑 + 一個交接點**。A 機從起點直走 3 步到交接點停下、舉手喊「換你」→ B 機收到後直走 3 步到終點 → 兩機舉手勝利。
- 先把「移動 + 交接點停下 + 手勢交棒 + 同步勝利」這條主幹跑通,障礙避障與多轉彎關卡可作為後續加分迭代。

**展演高光時刻(現場 6 分鐘最吸睛橋段)**
- 第 0-3 分鐘:A 機真的沿著地板膠帶往前走,走到交接點「啪」地停下、**舉起手臂喊『換你』**——觀眾視線從 A 轉到待命的 B。
- 第 3-5 分鐘:B 機**啟動接力**,在地板上繞過實體障礙、走到寶物點。
- 第 5-6 分鐘:兩台 Kebbi 在終點**同步舉雙手做勝利手勢**+ 音效,全場圍觀同一物理時刻——這是平板永遠演不出來的「兩個會動的實體在你腳邊合作完成任務」的臨場高光。

#### 〔互動遊戲〕誰在叫我?Kebbi 轉頭找人點名大挑戰

**對象與發展適配（為何適合這組年齡與程式深度）**
- 適合小五升小六（10-11 歲）：此年齡正值大肌肉協調與空間方位感成熟期，「圍著實體繞圈、聽口令快速換位」的全身性玩法能滿足好動需求，又比低年級更能理解「方位、回合制、計分博弈」等抽象規則。
- 程式深度恰到好處：學生需設計「狀態機」（倒數→偵測→判定→計分→加速關卡），導入聲源方位的條件分支與閾值判斷、頭部/手臂角度對應方位的參數表、隨關卡遞減的倒數時間變數。這對高年級是「有挑戰但做得完」的甜蜜點，可分組各自負責流程、動作、計分模組，培養模組化與整合能力。
- 體育與資訊跨域：把「鬼抓人／躲貓貓」的傳統遊戲重構成可程式化的人機對戰，貼近此年齡的生活經驗，學習動機高。

**真實需求/使用者故事**
- 一個想練習反應力與專注力的學生，希望有個「會回應方向」的玩伴；但獨自一人時，沒有人能陪他玩需要躲藏與聽聲辨位的遊戲。Kebbi 補上了這個「會聽、會轉身、會指人」的對手角色，讓單人也能玩到需要對手的空間躲避遊戲。
- 延伸需求：班級團康、課後等待時段，需要一個能公平主持「點名／找人」遊戲的非人類裁判。

**Kebbi 功能組合（標出平板沒有的能力，需要台數與分工）**
- 需要 Kebbi 台數：**1 台**。
- **聲源定位（方向性麥克風）＋轉向【平板做不到】**：核心機制。用方向性麥克風判斷「我在這」口令來自前／左／右／後方，並立刻把頭與身體轉向聲音來源。尤其「正後方」的判定，是單一平板麥克風物理上無法可靠分辨的關鍵差異。
- **關節手勢（頭／雙手）【平板做不到】**：轉向後用手臂在三維空間中「指向」喊聲的學生，提供被指認的身體性回饋；指對／指錯觸發計分。
- **實體在場【平板做不到】**：一個會轉頭、會轉身、會抬手指人的實體對手，產生「它真的在追我」的臨場壓力，這是螢幕箭頭無法製造的。
- 平板原生即可、不算特色的部分（僅作輔助）：TTS 報數與口白、喇叭播音效、ASR 辨識口令關鍵字、螢幕顯示比分。這些都不是核心，拿掉聲源定位與轉向後遊戲即垮。

**運作流程（分步驟，具體可實作）**
1. 開場：Kebbi 用 TTS 說明規則，學生站到起始方位。Kebbi 低頭「閉眼」（頭部下垂＋眼睛動畫）開始倒數（如 5 秒）。
2. 移動：倒數期間學生快速繞到任一方位（左／右／後方），在倒數結束瞬間喊「我在這！」。
3. 偵測：Kebbi 以方向性麥克風取得聲源到達方向（DoA），判定方位落在前／左／右／後哪個扇區。
4. 轉向＋指認：Kebbi 把身體與頭轉到該方位，並抬起對應手臂指向該方向，TTS 報出「你在我的左邊！」。
5. 判定計分：若 Kebbi 指對 → 學生「被抓到」，需立即換位（學生本回合扣分／Kebbi 加分）；若指錯（學生已成功騙過）→ Kebbi 失分、學生得「騙過」一次。
6. 關卡升級：每過一關縮短倒數時間（5→4→3 秒）、可加入「假口令」誘餌（學生先在 A 方位出聲再衝到 B 方位）增加難度。
7. 結算：累計回合後，TTS 宣布「你騙過我 N 次！」並用手勢做出勝負反應（鞠躬／舉雙手歡呼）。

**為何平板做不到-核心論證（差異化關鍵）**
- **可靠的 360 度空間定位（含正後方）**：單一平板僅 1-2 顆麥克風，無法穩定分辨「前／後／左／右」，正後方幾乎不可能；消費級裝置的麥克風陣列即便有波束成形，其「到達方向（DoA）」資料並不對開發者開放。要在純平板 App 上達成等同效果需外接麥克風陣列，已超出「平板 App」範疇，且仍不及機器人原生方向麥克風可靠。本作品明確要求學生「繞到後方」，平板在此直接失效。
- **真實的頭部／身體轉向**：平板是固定平面螢幕，當玩家繞到它背後時，螢幕上的「臉轉向」背後的人根本看不到；Kebbi 能把頭與身體真實轉 180 度面向背後的玩家，這是平板物理上做不到的臨場感與遊戲公平性來源。
- **手臂指向的身體性回饋**：平板沒有關節手臂能在 3D 空間實際指向玩家，只能畫螢幕箭頭。但箭頭要求玩家盯著螢幕，這與「繞到裝置背後再快速躲到下一方位」的繞圈走位機制直接衝突——玩家根本看不到也不該看螢幕。
- 結論：本作品的張力（看誰能在實體轉到面向你之前先躲開）本質綁定在「3D 空間移動＋會物理轉向的關節身體」。少了 Kebbi，遊戲從「躲避一個會轉身指人的身體」退化成「盯螢幕箭頭猜方位」，玩法與成效皆不成立。兩位平板律師均判定「可被平板取代＝false」。

**對應評分亮點**
- 應用性：把傳統「躲貓貓／點名」重構為可獨自遊玩的人機對戰，並可延伸為班級團康、反應力與專注力訓練教具。
- 創意性：以「聲源定位＋實體轉向＋手臂指認」組成「會聽聲找人的對手」，玩法與機器人原生能力高度綁定，非平板套殼。
- 挑戰性：聲源方位扇區判定、方位到頭/手角度的映射、隨關卡遞減倒數的狀態機、誘餌假口令博弈，整合難度足夠。
- 完整性：從開場規則、回合循環、關卡升級到結算與勝負動作，形成完整可重玩的遊戲閉環。
- 展演：圍繞實體繞行、被真實轉頭與抬手指認的瞬間具備強烈現場張力，6 分鐘易出高光。

**挑戰度（★1 到 5）與程式深度**
- 挑戰度：★★★★☆（4）。
- 程式深度：中高。需處理方向性麥克風 DoA 取值與扇區分類、ASR 口令觸發時序、方位→頭部/手臂角度對照表、隨關卡變化的倒數時間與誘餌邏輯、回合計分狀態機與結算。建議分為「感測判定」「動作映射」「遊戲流程/計分」三模組分工開發。

**可現場 demo 的 MVP（先做哪一條最小流程）**
- 最小流程：固定一關、倒數固定秒數 →（左／右／後三方位其一）學生喊「我在這」→ Kebbi 偵測方位 → 轉身＋抬手指向該方位 → TTS 報方位並判定指對/指錯一次。
- 先把「聲源方位 → 轉身指認」這條閉環做穩（重點驗證「後方」可被偵測並轉身面向），再疊加多回合、加速倒數、假口令誘餌與計分結算。

**展演高光時刻（現場 6 分鐘最吸睛橋段）**
- 高光：表演者悄悄繞到 Kebbi **正後方**喊「我在這！」，Kebbi 隨即把頭與整個身體轉 180 度面向背後的人並抬手「指中」——這一刻最能直觀展示「平板永遠看不到背後、也轉不過去」，全場感受到它「真的在找你」。
- 收尾：連續加速關卡下，表演者用「假口令誘餌」成功騙 Kebbi 指錯方向，Kebbi 做出搞笑「撲空」反應與結算歡呼動作，製造笑點與記憶點。

#### 〔生活應用〕藏寶圖尋寶王:會走路的線索精靈

**對象與發展適配(為何適合這組年齡與程式深度)**
- 小五升小六(10-11歲)正處於 Piaget「具體運思晚期 → 形式運思初期」,能處理空間方位推理、多步驟條件邏輯,但仍高度需要身體動覺與真實情境支撐學習,純螢幕容易久坐疲乏。本作以「離開螢幕、起身在房間奔跑解謎」為核心,正中此年齡層的運動需求與自主探索慾。
- 程式深度落在中階偏上:涉及「移動定點導航 + 關節手勢序列 + 聲源定位轉向 + 多步驟關卡狀態機」的協調,需要同學自行設計座標化的關卡地圖、定義手勢與謎題的對應表、處理「找對/找錯」的分支回饋,對高年級是有挑戰但可達成的甜蜜點。

**真實需求/使用者故事**
- 放學後一個人在家、功課寫完很無聊的小六生,渴望有個「會動的玩伴」陪他玩需要起身跑動的遊戲,而不是又盯著螢幕。痛點是「獨自在家 + 久坐螢幕 + 缺乏陪伴」,本作用一台會自己走、會比方向、會轉身歡呼的實體夥伴,把孩子拉離螢幕、在自家空間跑起來。

**Kebbi 功能組合(標出平板沒有的能力)**
- 需要 Kebbi 台數:**1 台**。
- **【平板沒有・移動】** Kebbi 自己沿規劃路徑「走到」房間各定點藏下「線索站」並躲在定點等待,孩子要找的是「會移動、會躲起來的它本身」,而非固定螢幕。關卡推進時 Kebbi 主動移動到下一站,形成空間中的躲貓貓追逐。
- **【平板沒有・關節手勢】** 用手臂「實際指向」下一站的真實物理方位(指左/指右/指前),用「舉手高度」表示距離遠近(越靠近舉越高),找對時「轉身 + 點頭 + 雙手歡呼」。這是物理方向示範,不是螢幕上的箭頭圖示。
- **【平板沒有・聲源定位 + 轉向】** 孩子在房間任意方位喊「Kebbi 我找到了!」或「我在這!」,Kebbi 用方向性麥克風判斷孩子在哪個方位,**主動轉身面向孩子**再給下一道謎題或回饋——孩子不必走回 Kebbi 面前,Kebbi 會「找到並面向」正在說話的孩子。這是本作補強物理依賴的關鍵新增機制。
- 語音出謎題、TTS/ASR 屬平板也有的能力,不列為特色,僅作輔助層。

**運作流程(分步驟,具體可實作)**
1. 設定階段:孩子帶 Kebbi 巡一圈房間,在 3-4 個定點按鈕「記錄」當前位置(里程計 + 轉角記錄成相對座標路徑),並錄入每站的謎底物品(如「沙發抱枕下」)。
2. Kebbi 移動到第 1 站,躲定後播放謎題,並用手臂「指向」下一個答案物所在的真實方位,舉手高度提示距離。
3. 孩子起身跑去該方位尋找答案物,找到後在原地大喊回報。
4. Kebbi 透過聲源定位判斷孩子方位、轉身面向孩子,確認對錯:對→點頭歡呼;錯→搖頭並重新比一次方向(可微調手臂角度當作「更接近/更遠」提示)。
5. 答對後 Kebbi 移動到下一站,重複 2-4,逐步推進。
6. 全破關後 Kebbi 在房間中央「轉圈 + 跳勝利舞 + 雙手高舉」播放勝利音效,並轉向孩子點頭。

**為何平板做不到-核心論證(差異化關鍵)**
- 平板架在腳架上「不會移動、不會躲」:本作核心是「線索精靈自己走到下一站藏起來,孩子要找的是會跑、會躲的它本身」,平板無法離開腳架去藏、去躲,躲貓貓追逐機制直接崩塌。
- 平板「沒有實體手臂」:用真實手臂指向房間中的物理方位、用舉手高度表達距離,是空間中的物理示範;螢幕只能顯示 AR 箭頭或唸方位詞,無法在三維實體空間用機械臂指出「就是那邊」。
- 補強的決勝點—**聲源定位轉向**:本版把「孩子在房間任意角落喊話 → Kebbi 判斷方位並主動轉身面向他」設為必經回報機制。固定腳架上的平板麥克風雖能收音,但**無法轉身朝向說話者**,孩子被迫走回螢幕正前方才能互動,這就把「在空間中自由奔跑」這個被宣稱的核心成效給破壞了。要在自家任意定點喊話、就被夥伴「轉頭找到並面向」,只有會移動會轉向的實體機器人辦得到。
- 三項物理能力(移動藏匿 + 手臂指向 + 聲源轉向)缺一,遊戲的「奔跑解謎 + 空間方位 + 自由回報」成效即無法成立,故為核心機制而非體驗加分。

**對應評分亮點**
- 應用性:解決小六放學後獨自久坐螢幕的真實痛點,用家中現成空間即可玩,落地門檻低。
- 創意性:把「藏寶圖」具象成「會走路會躲的線索精靈 + 手臂方向示範 + 喊話被轉頭找到」,將機器人三大物理能力編織成一個完整遊戲。
- 挑戰性:相對座標路徑導航、手勢距離編碼、聲源定位轉向、多關卡狀態機四者整合。
- 完整性:從設定 → 闖關 → 找錯回饋 → 破關勝利舞,具完整閉環與錯誤處理分支。
- 展演:會走、會比、會轉身歡呼跳舞,現場視覺與情感張力強。

**挑戰度與程式深度**
- 挑戰度:**★★★★☆(4)**。
- 程式深度:中階偏上。需里程計/轉角相對座標導航、關節手勢序列與距離編碼、聲源定位 + 轉向控制、關卡狀態機與對錯分支,並整合語音輸出。

**可現場 demo 的 MVP(先做哪一條最小流程)**
- 最小流程:單一關卡完整跑通——Kebbi 移動到第 1 站躲定 → 出謎題 + 手臂指向下一方位 → 評審/孩子跑去指定方位、喊「我找到了」→ Kebbi 聲源定位轉身面向他 → 點頭歡呼 → 移動到第 2 站。先確保「移動 + 手臂指向 + 聲源轉向 + 點頭」這條鏈穩定,再擴充到 3-4 站與勝利舞。

**展演高光時刻(現場 6 分鐘最吸睛橋段)**
- 高光一:Kebbi 自己「走」到房間另一角躲好、抬起手臂「指向」評審座位方向,孩子奔跑衝過去找線索——機器人移動 + 手臂指向同時發生,畫面立刻區隔於平板。
- 高光二(收尾殺手鐧):孩子故意跑到舞台側邊大喊「Kebbi 我找到啦!」,Kebbi **聽聲轉身、精準面向孩子**點頭歡呼,接著轉圈跳勝利舞、雙手高舉——「會聽方向、會轉身找人、會跳舞」三連擊,展示平板腳架永遠做不到的物理臨場感。
