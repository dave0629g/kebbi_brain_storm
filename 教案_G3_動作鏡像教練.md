# 初賽教案草稿 v1｜G3《Kebbi 體育課：動作鏡像教練帶你練分解動作》

> **草稿 v1**（依建議主攻；待確認後微調）。大會初賽 9 項格式、≤3 頁。評分：應用性 35／創意性 35／挑戰性 30。
> 遊戲邏輯已實作自測：`robot_app` 的 `MirrorCoachGame`（`--g3`/`--g3r`/`--g3f`）。

## 1. 科目領域
健康與體育（動作技能分解、節奏）為主，融入資訊科技（Python 入門／串接 API）。

## 2. 教學對象
國中三年級升高中一年級（14–15 歲），程式深度：Python 入門 / 串接 API 概念。

## 3. 學習目標
- 能將一個複合動作分解為「逐幀（key frame）」並掌握各幀的關節角度與停留節奏。
- 理解「鏡像對映」（教練面對學生時「我的左＝你的右」）並能據此調整示範。
- 能用「再一次／太快了」等回饋語調控學習節奏，培養自我監控（metacognition）。

## 4. 預期成果
Kebbi 用**真實上肢／頭部關節**做「可繞著看的立體鏡像示範」（暖身／太極／CPR 衛教姿勢）；聽到學生喊「**太快了**」會用聲源定位**轉頭面向發問者**並降低 BPM；喊「**再一次**」會**逐幀回退**重新示範該動作。

## 5. 對應課綱核心素養
- **健體**：動作技能的分解學習與節奏控制；健康衛教（CPR 姿勢概念）。
- **資訊科技**：以資料（逐幀＋停留時間＋循環次數）描述動作序列。
- **B1／A1**：身體感知與自我精進。

## 6. 課程架構圖
```
第1節 逐幀分解 → 第2節 鏡像示範 → 第3節 節奏與回饋 → 第4節 動作資料化+反思
 key frame      我的左=你的右      太快了→轉頭降BPM      HoldMs/循環/再一次回退
```

## 7. Kebbi 教案設計（核心；標出「只有機器人做得到」）
**器材**：1 台 Kebbi（可繞看的立體示範）、選配攝影機姿態檢查（可退人工確認）。

- **第 1 節・逐幀分解**：把「上肢開合／太極起式」拆成數個 key frame，學生跟著定格練。
- **第 2 節・鏡像示範**：Kebbi 用**真實關節**做立體示範，可繞到側面看清楚；示範「鏡像版」（左右對調，中軸頭部不換）對應「面對面」教學。〔**平板做不到**：立體、可繞看、真實關節的肢體示範——螢幕只有平面影片〕
- **第 3 節・節奏與即時回饋**：學生喊「太快了」→ Kebbi 用**方向性麥克風定位**並**轉頭面向發問者**、把 BPM 降一級。〔**平板做不到**：聲源定位 + 轉頭面對說話者〕
- **第 4 節・動作資料化＋反思**：示範「動作 = 逐幀資料」：某幀可設較長停留（如 CPR 下壓刻意停 2 秒）、整組可循環 N 次；喊「再一次」逐幀回退重練。學生嘗試自訂一組動作的節奏。
- **誠實邊界**：只示範上肢／頭部（Kebbi 無下肢），落在硬體優勢內。
- **差異化**：入門→跟做內建課表；進階→用資料自編節奏／循環。

**對應評分**：應用性（體育／衛教課可用）／創意性（立體鏡像＋聲源轉頭調節奏）／挑戰性（逐幀資料化＋鏡像對映＋回饋控制）。

## 8. 教學成果與回饋
- 形成性：姿態檢查（相機或人工）即時計分；「再一次」回退重練降低挫折。
- 總結性：能否完成整套課表並自訂一組動作的逐幀節奏。
- 預期回饋：可繞看的立體示範 + 轉頭互動，比平面影片更易模仿、更有臨場感。
